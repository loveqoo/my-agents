# 013 — 매 응답 `[단계 N — 이름]` 표기를 빠뜨리지 말 것

날짜: 2026-06-23
유형: 절차(자기 교정)

## 반복한 실수
CLAUDE.md "빠른 합의 규칙": **매 응답 머리에 현재 6단계 위치를 `[단계 N — 이름]`로 표기**.
초반 루프엔 지켰으나, 자율 실행(007~008, 원격, env 제거)에 몰입하면서 **대부분 응답에서 누락**했다.
사용자가 "6원칙 지켜라, 실수 반복 말자"로 지적 → 명백한 반복 lapse.

## 왜 중요한가
- 사람과 AI가 **지금 루프 어디인지 공유**하는 장치. 빠지면 단계 합의·스킵 사유가 흐려진다.
- 자율 위임이라도 절차는 유지해야 "선명한 절차"라는 저장소 목표가 지켜진다.

## 교정 규칙 (앞으로)
- **모든 응답 첫 줄에 `[단계 N — 이름]`**. 자율 모드여도 예외 없음.
- 한 응답이 여러 단계를 걸치면 주된 단계를 표기(필요시 전환을 본문에 명시).
- 기능 루프마다 **Compounding에서 회고 1건**(`.dev/retrospect/NNN`) 또는 최소 학습 1건 — 학습만으로 끝내지 말고 루프 단위 회고를 남길지 판단.

## 안 한 것 (유지 중인 것)
플랜=파일(spec), 검증=타자(codex), 학습 자산화는 계속 지켜짐.
