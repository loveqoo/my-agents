# 093 — 값에 의존하는 effect는 값이 안 변하면 안 돈다 / 비동기 재계산 리스트에 보존한 인덱스는 경합

> 스펙 091(플레이그라운드 입력 히스토리 ↑/↓). 적대 codex가 P2 2건을 짚었고, 둘 다 *happy-path 단위·브라우저
> 초록*을 통과한 채 숨어 있었다. 둘의 공통 뿌리: **"이벤트가 일어났다 ≠ 관찰 가능한 값이 바뀌었다"**.

## 언제

- **명령형 DOM 부수효과(caret 이동·포커스·스크롤)를 React `useEffect`/`useLayoutEffect`로 걸 때.** 특히
  "사용자 액션마다 한 번씩" 돌려야 하는데 의존 배열에 *결과값*(`[draft]`/`[value]`)을 넣었을 때.
- **`useMemo`/쿼리로 비동기 재계산되는 리스트에, 외부 액션이 보존한 인덱스(`ref.current.idx`)로 접근할 때.**
  탐색 중 소스(`messages`)가 바뀌어 리스트 길이가 줄면 그 인덱스가 범위를 벗어난다.

## 무엇

### P2-1. 값 의존 effect는 no-op setState에서 안 돈다

`useLayoutEffect(() => { moveCaretToEnd() }, [draft])`로 caret을 끝으로 옮겼다. 그런데 **재호출 값이 현재
입력과 같으면**(예: 최신 입력이 이미 입력창에 그대로) `setDraft(sameValue)`는 **no-op → 리렌더 없음 →
effect 미발화**. caret이 안 옮겨질 뿐 아니라, 발화를 한 번 더 트리거하려고 둔 `caretEndRef=true`가 **소비
안 된 채 남아** *다음 진짜 편집*에서 caret이 엉뚱하게 끝으로 튄다(상태 누수까지).

- **처방**: effect 의존을 결과값이 아니라 **단조 카운터**로. 액션마다 `setSeq(n => n+1)`, effect deps `[seq]`.
  카운터는 매번 *반드시* 변하므로 값 동일성과 무관히 발화한다. 보조 ref 플래그(`caretEndRef`)는 제거 —
  "한 번 더 발화시키려는" 플래그 자체가 누수원이었다.
- **일반화**: "이벤트마다 발화"가 필요하면 **이벤트 카운터에 의존**시켜라. 값에 의존시키면 React의
  *값 동일성 단락*(`Object.is`로 같으면 리렌더 스킵)이 happy-path엔 안 보이는 구멍을 판다.

### P2-2. 비동기 재계산 리스트에 stale 인덱스로 접근하면 undefined가 샌다

탐색 상태 `histRef.idx`는 키 핸들러가 보존하지만, `history = useMemo(..., [messages])`는 메시지가
바뀔 때마다 새로 계산된다. 탐색 중 history가 줄면 `idx`는 *옛 길이* 기준이라, `recallNewer`가
`history[len-1-idx]`에서 **음수 인덱스 → `undefined`**를 controlled `draft`로 흘려 **입력창이 빈다**.

- **처방**: 접근부(리듀서)에서 **현재 길이로 먼저 클램프**: `cur = Math.min(idx, len-1)`. 빈 리스트면
  초안 복원·탐색 종료로 흡수(`undefined` 금지). 반대 방향(`recallOlder`)은 이미 `Math.min(idx+1, len-1)`
  클램프라 안전했다 — *한 방향만 막으면 반대 방향이 샌다*.
- **일반화**: **외부에 보존한 인덱스 + 비동기로 재계산되는 리스트 = 항상 경합.** 접근 시점에 현재 길이로
  클램프하거나 스냅샷을 같이 보존하라. "useMemo는 동기처럼 보인다"가 함정 — 의존이 바뀌면 *다음 렌더에*
  바뀌고, 그 사이 보존 인덱스는 stale.

## 검증에도 같은 함정 (메타)

브라우저 회귀 테스트의 **셋업**이 같은 이유로 처음 실패했다: 직전 단계가 navigating 상태를 남겼는데
**Playwright `fill(동일값)`은 onChange 미발화**라 탐색 리셋이 안 됐다. `fill('')→fill(값)`으로 강제 리셋.
제품 버그가 아니라 테스트 artifact였지만 뿌리는 P2-1과 같다 — **값이 안 바뀌면 변경 이벤트가 안 온다.**

## 적용 규칙

1. **명령형 DOM 부수효과를 effect로 걸 땐 의존 배열을 의심하라.** 값 의존이면 자문: "이 값이 *안 변할*
   때도 이 effect가 돌아야 하나?" 그렇다면 → 단조 카운터. 보조 ref 플래그로 "한 번 더 발화"를 흉내내지 말 것
   (그 플래그가 미소비로 남아 다음 사이클을 오염시킨다).
2. **비동기 재계산 리스트에 보존 인덱스로 접근하면 접근부에서 클램프**(`Math.min(idx, len-1)`), 양방향 둘 다.
3. **"이벤트 발생 ≠ 관찰값 변경"** — 값 동일성에 기댄 어떤 메커니즘(effect deps·onChange·fill)도
   값이 같으면 단락된다. happy-path 단위·브라우저 초록이 이걸 못 잡으니 **적대 검증으로 여집합을 시켜라**.
