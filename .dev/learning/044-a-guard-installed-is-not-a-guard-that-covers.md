# 044 — "안전장치를 깔았다"는 "안전하다"의 증거가 아니다 — 적용 범위 경계를 적대자가 짚는다

## 상황

A2A liveness probe에 SSRF 가드(`guard_url`)를 깔았다. 그 사실이 안심시켜 같은 줄의
`follow_redirects=True`를 못 봤다. 가드는 **최초 URL의 resolve IP만** 검사한다 — 공개 호스트가
302로 `169.254.169.254`나 `127.0.0.1`을 가리키면 httpx가 추종해 가드를 그대로 우회한다(블라인드
SSRF 도달성 오라클). 적대 서브에이전트가 잡았다. 내 단위 테스트는 httpx를 목으로 막아 *구조적으로*
이 구멍을 볼 수 없었다(목은 리다이렉트하지 않는다).

## 교훈

**보안 장치를 "설치했다"는 사실과 그 장치가 "공격 표면 전체를 덮는다"는 사실은 다르다. 적대자는
장치의 *적용 범위 경계*를 짚는다 — 가드가 검사하는 지점(최초 URL)과 실제 요청이 도달하는 지점
(리다이렉트 체인 끝)이 어긋나는 틈.**

- 한 번 검사(getaddrinfo) 후 다른 주체(httpx)가 재해석/추종하면 TOCTOU·우회가 열린다.
- 단위 테스트가 외부 의존(httpx)을 목으로 막으면, 그 *목이 표현하지 않는 동작*(리다이렉트,
  DNS 재바인딩)은 테스트가 영원히 못 본다. "단위 그린"이 "경계 안전"을 뜻하지 않는다.
- 방어: 장치의 *적용 경계 자체를 단언*하는 테스트(예: `AsyncClient(follow_redirects=False)`인지
  생성 kwargs를 캡처해 검사 — 행동이 아니라 *설정*을 못 박는다). 그리고 가드 적용 후 *상태를
  바꾸는 옵션*(redirect 추종, 재해석)을 끈다.

learning 038("적대 리뷰는 상상 못 한 범주를 찾는다")의 보안판 따름정리. 038은 "내 불변식이 못 본
것", 044는 "내 *안전장치*가 못 덮는 것" — 더 위험한 건 장치의 존재가 주는 거짓 안심이다.

## 적용 트리거

검사하는 지점과 실제 부수효과가 일어나는 지점이 같은가? 사이에 누가 재해석·추종하는가?"를 묻는다.
