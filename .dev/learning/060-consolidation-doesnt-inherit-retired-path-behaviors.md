# 060 — 통합은 폐기 경로의 행동을 자동 승계하지 않는다: 살아남는 경로에 폐기 경로의 *행동 집합*을 열거해 이식하라

## 언제
두 코드 경로(자체 프로토콜 vs 표준, 레거시 핸들러 vs 신규, 분기 A vs B)를 **하나로 합치면서 한쪽을
삭제**할 때. 특히 "표준으로 통일한다"·"중복 제거한다"처럼 살아남는 경로가 *더 옳아 보일* 때.

## 명제
**삭제는 폐기 경로가 *고유하게* 들고 있던 행동을 함께 지운다.** 살아남는 경로가 컴파일되고 happy-path가
초록이어도, 폐기 경로만 하던 일은 조용히 사라진다 — 그 행동은 통합 diff 어디에도 나타나지 않기 때문에
(빠진 코드는 보이지 않는다) 자가검증이 구조적으로 못 본다. "두 경로를 하나로"는 합집합이 아니라
기본적으로 *교집합*(둘 다 하던 일)으로 수렴하고, 차집합(한쪽만 하던 일)은 누락된다.

## 표본 (스펙 057 · A2A 단일화)
자체 SSE `_remote_stream`을 폐기하고 모든 원격 에이전트를 표준 `_a2a_stream` 하나로 합쳤다. 컴파일·
단턴 채팅 모두 초록. 그러나 `_remote_stream`은 윈도우 히스토리를 **인라인 전송**했는데 `_a2a_stream`은
마지막 메시지만·`contextId` 없이 보내 code 에이전트가 **단턴화**(멀티턴 맥락 상실). diff엔 "삭제"만
찍혀 그 행동이 안 보였고, codex 적대 리뷰(타자)가 잡았다. → A2A 표준대로 세션 id를 `message.contextId`로
실어 서버가 맥락 유지, 회귀가드로 음성 인코딩(`_jsonrpc_request(context_id=...)`·`_a2a_stream`이
`session_id`를 contextId로 싣는지 소스 단언).

## 처방
- **삭제 전에 폐기 경로의 행동을 열거하라**: 입력 변환(이 경우 히스토리 인라인), 부수효과, 기본 헤더/
  옵션, 에러 프레이밍 — "이 경로만 하던 일"을 목록화하고 살아남는 경로가 각각을 어떻게 흡수하는지 명시.
  흡수 못 하는 항목은 회귀다.
- **표준이 더 옳아 보여도 *기능 패리티*를 먼저 확인**: 표준 경로의 happy-path 통과는 "폐기 경로 대체
  완료"가 아니다. 폐기 경로가 풀던 *문제*(멀티턴·재시도·캐싱 등)를 표준 경로가 푸는지 별도로 단언.
- **회귀가드는 "행동의 *부재*"를 단언**: 빠진 코드는 안 보이므로, "contextId가 실린다" 같은 *있어야 할
  행동*을 소스/런타임에서 직접 단언(051·055의 "도구 *부재* 단언" 쌍대 — 여기선 행동 *존재* 단언).
- **타자 적대 검증 필수**: 누락은 내 머릿속 행동 모델의 여집합이라 자가검증이 못 밟는다. codex/서브에이전트에
  "폐기 경로가 하던 일 중 살아남은 경로가 안 하는 것"을 명시적으로 시켜라.

## 친척
- `move-breaks-references-both-directions`(이동/삭제가 양방향으로 깸)의 *행동* 판 — 그건 참조 좌표,
  이건 행동 좌표. 둘 다 "사라진 것은 안 보인다".
- `055`(새 아웃바운드 경로는 가드 자동상속 안 함)의 거울상 — 거긴 *새* 경로가 안전을 상속 못 함,
  여긴 *통합된* 경로가 행동을 상속 못 함. 공통 뿌리: 경로 위상이 바뀌면 보장/행동은 자동 따라오지 않는다.
- `adversarial-review-before-destructive-ship`·`verification-ladder-three-rungs`: 통합·삭제는 비가역
  파괴 경로라 적대 타자가 상상 못한 누락을 잡는다(rung3). 라이브 통합(rung2)이 seed drift를 별도로 잡음.
