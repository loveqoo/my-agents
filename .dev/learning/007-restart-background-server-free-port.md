# 007 — 백그라운드 서버 재기동 시 포트 점유를 확실히 정리

날짜: 2026-06-19
type: reference

## 무엇이 문제였나
- API(uvicorn)를 수정 후 `pkill`만으로 종료하고 재기동했더니 새 프로세스가
  `[Errno 48] address already in use (127.0.0.1:8000)`로 **기동 실패**.
- 옛 프로세스가 포트를 계속 쥐고 있어, **옛 코드가 응답** → 수정(CORS 127.0.0.1)이
  반영 안 된 것처럼 보였다(127.0.0.1 CORS 헤더 누락으로 발각).

## 그래서 규칙
- 백그라운드 서버 재기동 시 **포트 점유자를 직접 정리**한다:
  `lsof -ti tcp:<PORT> | xargs kill -9`
- 재기동 후 **변경이 실제 반영됐는지 검증**한다(예: 새 CORS 오리진 헤더 확인).
  "응답이 온다"는 옛 프로세스일 수 있으니 동작이 아니라 *변경점*을 확인.

## 적용
- reload 없는 서버(`uvicorn` 직접 실행 등)는 코드 변경 후 반드시 종료→포트확인→재기동.
- 가능하면 개발 중 `--reload` 사용 고려(단, 본 프로젝트는 단순 실행 유지).
