# 086 — 소비층 fail-closed만으론 "죽은 상태"가 샌다: 술어를 *모든 쓰기 경계*에 정렬하라

## 신호 (언제 이 learning을 떠올리나)

- 어떤 *capability 플래그*(노출/활성/공개/허용)가 **한 층(소비측)에서만 fail-closed**로 강제되고 있다.
- 사용자가 보안 결함이 아니라 "이거 이상한데/되는 것처럼 보이는데 안 됨"을 보고한다.
- 플래그를 *쓰는* 입구(API·시드·기존 DB·mock)와 *그리는* 입구(드로어·테이블·배지)가 여럿이다.

## 핵심

소비측을 fail-closed로 닫는 건 **동작/보안**을 지키지만 **데이터 정합·UX**는 못 지킨다. 플래그를 *쓰는·그리는*
입구가 같은 술어를 공유하지 않으면 **"겉도는 죽은 상태"(dead state)** — 플래그는 켜졌으나 소비층이 무시해
아무 일도 안 일어나는데 UI는 켤 수 있다고 말하는 상태 — 가 쌓인다. 이건:

- **happy-path·보안 테스트에 안 잡힌다** (소비층이 거부하니 "동작은 맞음"). 사용자가 먼저 "이상함"으로 짚는다.
- 고치려면 **소비층 게이트의 술어(예: `source != "ui"`)를 모든 쓰기/렌더 경계에 그대로 복제**해야 한다:
  입구(거부)·시드·기존 데이터 마이그레이션·UI 렌더·중앙 헬퍼·배지/카운트. 한 곳이라도 빠지면 그 입구가
  죽은 상태의 새 생산지가 된다([[probe-deeper-before-concluding]]: "서버가 막으니 됐다"에서 한 겹 더).

## 처방

1. **불변식을 한 줄로 적는다**: `flag==True ⟹ predicate`. 이게 *모든 입구*가 지켜야 할 계약.
2. **입구를 닫힌 집합으로 센다**(RBAC 체크리스트의 입구 열거와 같은 규율): write-API / seed / 기존 데이터 /
   mock / 렌더(드로어·테이블) / 배지·카운트 / 중앙 토글 헬퍼. 주 경로 하나만 보면 샌다.
3. **끄기(False)는 술어 무관 항상 허용** — stale 플래그를 멱등으로 청소하는 경로를 막지 마라.
4. **프런트 숨김에 의존 금지**(learning 027) — 입구에서 거부(400)하고, 렌더 숨김은 *추가* 층으로만.
5. **기존 데이터도 입구다** — 마이그레이션으로 stale을 청소해 불변식을 *데이터*에 성립시킨다(정상 값은 보존).

## 곁가지 (이번에 codex가 덤으로 짚은 것)

JSONB 컬럼을 갱신할 때 **통째 교체(`= {"k": v}` / `'{"k":v}'::jsonb`)는 형제 키를 파괴**한다. "지금은 키가
하나뿐"이라도 merge(`{**(col or {}), ...}`) / `jsonb_set(col, '{k}', v, true)`로 쓰고 *형제 키 보존 단언*으로
잠가라. "오늘 단일 키"는 내일 깨질 가정이고 하드닝은 1줄이다. ([[adversarial-review-before-destructive-ship]])
