# 012 — 런타임 설정은 단일 소스(레지스트리)에서, env는 부트스트랩만

날짜: 2026-06-23
맥락: [008 모델 레지스트리](../../docs/spec/008-model-registry.md), `build_agent`/`chat.py`/`memory.py`

## 결정
LLM/임베딩 설정을 **모델 레지스트리(DB) 한 곳**에서만 해석한다. 런타임 코드(`build_agent`,
`chat.py`, `memory.py`)는 **env(MLX_*)를 보지 않는다**.
- `build_agent`: `model_cfg` 필수(base_url/model_id). 없으면 에러.
- `chat.py`: 에이전트가 고른 모델 → 레지스트리, 없으면 **기본(is_default) chat 모델**, 그것도 없으면 400.
- `memory.py`(mem0): LLM·임베딩을 호출자가 넘긴 레지스트리 설정으로 구성.
- **env는 두 군데만**: `seed.py`(첫 부팅 시 레지스트리 부트스트랩), 에이전트 **CLI**(레지스트리 없는 단독 도구). 둘 다 런타임 경로가 아님.

## 왜
- 폴백 env가 있으면 "등록 안 된 모델로도 어떻게든 돈다" → 레지스트리가 진실의 소스가 아니게 되고, 어떤 모델로 실행됐는지 불명확. 폴백 제거로 **실행=등록된 설정**이 보장됨.

## codex가 잡은 함정 (반영)
- **캐시 키에 자격증명 포함**: mem0 인스턴스를 `(base_url, model_id)`로만 캐시하면 **api_key 회전 후 stale 인스턴스**가 재사용된다. 키에 api_key까지 포함(메모리 내, 미로깅).
- **불완전 설정은 400으로**: 빈 base_url/model_id 모델을 고르면 `build_agent`가 RuntimeError(→500). `_load_context`에서 미리 검증해 **400**으로.
- **kind별 단일 기본값**: `is_default` 모델이 여러 개면 `.first()`가 임의 선택. 생성/수정 시 같은 kind의 다른 기본값을 끈다.

## 교훈
설정은 한 소스로 모으고, **폴백을 지워 소스를 강제**한다. 자격증명으로 만든 캐시는 **자격증명을 키에** 넣는다.
