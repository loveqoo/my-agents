# 022 — 뷰포트 브레이크포인트는 옆 패널 폭을 모른다

`Grid.useBreakpoint()`(antd) / CSS 미디어쿼리는 **뷰포트 전체 폭**을 잰다.
같은 행에 고정폭 형제(사이드바·인스펙터·드로어)가 있으면 그 폭이 빠진
**콘텐츠 영역 잔여 폭**과 뷰포트 폭은 다르다. 반응형 판단을 뷰포트 폭으로 하면
콘텐츠가 짜부라지는 폭을 놓친다.

## 사건 (Playground #9)
- 레이아웃: `사이드바(232) + 채팅(flex:1) + 인스펙터(384)` 한 행.
- `compact = !screens.lg`(992) — 뷰포트 992 미만에서만 헤더 축소.
- 1020px: 뷰포트는 992를 넘어 풀 라벨인데, 채팅 컬럼 실제 폭 = 1020−232−384 = **404px**.
  → 헤더 컨트롤이 인스펙터 헤더 위로 흘러넘쳐 "턴 인스펙터" 타이틀·아이콘과 겹침.
- 768~880px대에선 채팅 컬럼이 ~184px로 더 심하게 짜부라짐.

## 처방
1. **고정폭 패널이 켜져 있으면 그만큼 더 일찍 축소**한다.
   - 오버레이 임계값 상향: 인스펙터를 `md(768)`→`lg(992)` 미만에서 전체화면 오버레이로.
   - `const compact = !screens.lg || inspectorOpen` — 패널이 열린 동안은 폭과 무관하게 축소.
2. 더 정확히 하려면 뷰포트가 아니라 **컨테이너 폭**으로 분기(`ResizeObserver` / container query). 이번엔
   "패널 열림" 플래그로 충분해서 거기까진 안 갔다.

## 재현·검증
- 폭 스윕(760~1220px) + 스크린샷이 단일 폭 측정보다 강하다. UI 버그가 단일 폭에서 재현
  안 되면 **폭을 스윕**하라(측정과 사용자 눈이 어긋날 때 특히).
- 합성 `setViewportSize` 후 패널이 빈 상태로 보일 수 있다(리마운트 아티팩트) — 폭별로
  상태를 새로 수립해 확인.
