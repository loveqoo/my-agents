# 089 — 사람에게 보일 임의 데이터는 키 blocklist가 아니라 값 allowlist로(노출 deny-by-default)

**언제**: 신뢰할 수 없는/임의 구조(LangGraph 상태 델타, 외부 페이로드, 사용자 입력 dict)를 *사람이 볼
표면*(인스펙터·로그·trace·디버그 UI)에 요약·표시할 때. 비밀 누출 0이 불변식일 때 특히.

**무엇**: 비밀을 *키 이름 패턴*(`api_key`·`token`…)으로 마스킹하는 blocklist는 필요조건이지 충분조건이
아니다. 값 자체가 비밀인데 키가 평범(`note`·`memo`)하거나 비영문(`비밀번호`)이면 패턴이 못 잡아
값이 그대로 샌다. 자가 단위 테스트는 *민감한 이름의 키*만 확인해 전부 초록이고 — 적대자가 *평범한
이름의 비밀 값*을 짚는다(spec 086 codex F2). happy-path가 보는 건 내가 상상한 실패뿐.

**처방**:
1. **노출을 뒤집어라 — 값 allowlist(fail-closed)**. 값을 *원문으로* 보여줄 키는 "이 기능의 존재이유가
   그 값을 보이는 것"인 필드만(최소 집합). 그 외 임의 키의 문자열 값은 *크기/구조만*(`key: <N자>`·
   `key[N]`·`key{키목록}`). 키 blocklist는 그 위에 *추가로* 깐다(알려진 민감 이름은 즉시 마스킹).
2. **allowlist는 작을수록 옳다**. 편의로 `status`·`step`을 넣는 순간 "그 키는 절대 비밀이 아니다"를
   보증해야 한다 — 임의 에이전트가 거기 비밀을 담을 수 있으면 넣지 마라. 새 안전 키 추가는 그 보증과
   짝지어라(086은 `{plan, step, next, status, stage}`에서 `{plan}` 하나로 좁혔다).
3. **모든 형제 표면에 같은 정책을**. 한 표면(노드 summary)을 가려도 *옆 표면*(같은 trace의 MCP 호출
   인자/result)이 원문이면 같은 화면에서 샌다(086 F1=P0, 별 표면이라 087로 분리). 재사용 가능한
   redactor를 만들어 *모든* 표시 입구에 적용 — "입구를 닫힌집합으로 세라"(learning 070·088)의
   redaction판.
4. **fail-closed는 예외까지**. 비문자 키·미지 타입으로 요약이 *터지면* 표시 보조 기능이 본 스트림을
   깬다(086 F5). `str(key)` 정규화 + 요약기 전체 try/except → "실패=미표시(None)". 불확실하면 적게
   노출하고 죽지 마라.
5. **캡은 budgeted, 이중 캡 주의**. 값을 join *전에* 캡해 거대 값을 통째 만들지 마라([[cap-the-raw-source-not-the-buffer]]=059).
   단 필드 캡과 컨테이너 캡이 같으면 단일 큰 값이 *두 번* 잘려 생략 길이가 거짓이 된다 — 필드
   캡 < 노드 캡으로 분리(086: 300/1200).

**검증**: 적대자에게 "마스킹한 *이름* 말고, 평범한 이름의 *값*으로 새는 경로는?"을 시켜라(여집합).
회귀는 평범/비영문 키의 비밀 값·거대 값·비문자 키·형제 표면을 그린 단언으로 박아라(086 U1c·U2b·U4c).

**공명**: 089는 [[installed-guard-isnt-covering-guard]](가드 설치≠덮음)·[[adversarial-review-before-destructive-ship]](여집합)·
088(입구 닫힌집합)과 같은 결 — *판정 지점과 노출 지점이 어긋난 틈*. 086의 축은 "키 패턴 vs 값 자체",
대상은 "사람에게 보일 표면의 redaction". [[probe-deeper-before-concluding]]: 자가 초록을 "비밀 안 샌다"로
단정 말고 한 겹 더(평범한 키의 값).
