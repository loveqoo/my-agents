# 023 — 프로세스 규칙은 가시적 강제 산출물이 있어야 지켜진다

절차 규칙을 CLAUDE.md에 적어두는 것만으로는 지켜지지 않는다. 작업에 몰입하면 스킵하고,
**응답에 남는 가시적 산출물이 없으면 스킵이 검출되지 않아** 반복된다. 규칙은 "존재"가 아니라
"가시적 강제물"로 강제된다.

## 사례 (같은 메타 패턴)
- **1차** [[013-keep-the-step-header]]: `[단계 N — 이름]` 헤더 규칙이 있었으나 자율 실행 몰입 중
  대부분 응답에서 누락 → 사용자 적발.
- **2차** (스펙 021 후): Context 단계에서 learning/retrospect를 읽으라는 규칙이 있었으나
  *가시적 산출물*이 없어 상습 스킵 → 회고 복리 0 → 사용자 적발. 처방: Context 응답에
  **`참고 자산: [번호-제목 → 적용점]`** 필수 줄을 박아 스킵을 즉시 검출 가능하게.

## 처방 (재사용 원리)
1. **새 절차 규칙을 만들 때 "가시적 강제물"을 함께 설계**한다 — 응답에 남는 1줄/헤더/체크.
   규칙만 적고 끝내지 말 것. ("이걸 어떻게 눈에 보이게 강제하지?"를 규칙과 한 세트로.)
2. 가시적 산출물 = 사용자가 응답 머리에서 **스킵을 한눈에** 잡을 수 있는 형태(헤더, 라벨 줄).
3. 같은 lapse가 2회 반복되면 그건 "더 분발" 문제가 아니라 **장치 부재** 문제 → Scaffolding에서
   장치를 박는다(CLAUDE.md 규칙·훅·헤더). cf. CLAUDE.md "같은 실수 반복 → 훅이나 장치로 막는다".
