# 014 — 비밀값 at-rest 암호화(Fernet)와 함정

날짜: 2026-06-23
맥락: [010 비밀 암호화](../../docs/spec/010-secret-at-rest.md), `crypto.py`/`chat.py`/`model_registry.py`

## 패턴
- 비밀(모델 api_key, 코드 에이전트 token)은 **DB에 Fernet 암호문**으로 저장. 출력은 **고정 마스킹**(`••••••••`) — 부분 평문도 노출 안 함. 런타임에서만 복호화.
- 키 관리: env `APP_SECRET_KEY` → 없으면 `.dev/.secret_key`(생성·영속, gitignore). 개발 즉시 동작, 키는 코드/DB 밖. 운영은 env 주입.
- 경로 주의: repo 루트는 `crypto.py`(packages/api/src/api/) 기준 **parents[4]**. (parents[3]=packages 였던 버그)

## codex가 잡은 함정 (반영)
- **키 회전/불일치 은폐(치명)**: `decrypt`의 `InvalidToken` 폴백이 레거시 평문 + "다른 키로 암호화된 진짜 암호문"을 구분 못 함 → 키가 틀리면 **암호문을 그대로 반환** → LLM 키/Bearer로 외부 전송돼 **암호문 누출 + 설정오류 은폐**. → 값이 **Fernet 토큰 형태(`gAAAAA`로 시작)면 폴백 금지·에러**, 아니면 레거시 평문으로만 폴백.
- **에러 본문 에코 누출**: 원격 4xx 본문을 클라에 그대로 보내면 **Authorization/토큰을 에코**할 수 있음 → 본문은 **서버 로그로만**, 클라엔 상태코드만.
- **키 제거 의미 구분**: None/마스킹=보존, 빈문자열=제거, 그 외=새 평문 암호화. (보존만 하면 stale 자격증명 잔존)

## 교훈
"복호화 실패 시 입력 반환" 같은 무중단 폴백은 **비밀에선 위험** — 평문/암호문/잘못된키를 형태로 구분하고, 비밀은 실패를 **숨기지 말고 드러낸다**. 에러 메시지에 비밀이 새지 않게(본문 에코 금지).
