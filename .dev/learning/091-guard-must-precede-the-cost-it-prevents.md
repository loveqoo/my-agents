# 091 — 비싼 연산을 막는 가드는 그 연산 *앞*에 둬라(검사지점=비용지점 앞), 그리고 정직한 타입 생산자가 없으면 메타 파이프라인 말고 렌더시점 휴리스틱

> 언제: 거대/적대 입력으로부터 비싼 동기 연산(parse·재귀 렌더·직렬화)을 가드로 막을 때,
> 그리고 "타입을 받아 렌더러를 고르고 싶은데" 그 타입을 *정직하게 생산하는 주체가 없을* 때.

관련: [[installed-guard-isnt-covering-guard]](082 *적용범위* 축 — 이건 *비용지점 대비 순서* 축),
[[cap-the-raw-source-not-the-buffer]]=059(어디서 캡하나), [[core-is-model-config-and-memory]](보조도구 과한 금칠 금지),
[[adversarial-review-before-destructive-ship]]·[[probe-deeper-before-concluding]], retrospect 072.

## 사실 1 — 가드가 *비용 발생지점 뒤*에 있으면 설치돼도 안 덮는다 (순서 축)

플레이그라운드 JSON 트리에 "거대 JSON → 트리 렌더 프리즈"를 막으려 `jsonTooBigForTree`(길이 50k 캡)를
넣고 트리/원문 분기에서 호출했다. 자가로는 충분해 보였다. 그런데 codex가 짚었다:
**`detectFormat`이 `jsonTooBigForTree`를 보기 *전에* 이미 `JSON.parse(전체)`를 동기 실행**한다.
즉 길이캡은 *트리 노드 폭주*는 막지만, parse 자체의 비용은 가드 *앞에서* 이미 지불된다.

[[installed-guard-isnt-covering-guard]]가 *적용범위*(가드가 닿지 않는 입구)를 말했다면, 이건 같은
"non-covering"의 **시간/순서 축**이다: 가드의 *검사지점*이 그것이 막으려는 *비용지점보다 뒤*에 있으면,
가드는 설치돼 있어도 그 비용을 못 막는다. 처방은 단순하다 — **가드를 비용 발생지점 *앞*으로 옮긴다.**

단, 옮길 때 *모든* 비싼 경로를 덮어야 진짜 covering이다. 거대 입력은 `JSON.parse`만 비싼 게 아니라
markdown 경로의 remark 파싱도 비싸다 → 트리캡(50k)과 *별개로* 렌더 예산 상한(`exceedsRenderBudget`,
>1MB)을 `detectFormat`·markdown 디스패치 **둘 다보다 앞** 최상단에 둬, 어느 형식 추론보다 먼저
원문 캡 블록으로 직행시켰다. (트리캡 50k < 렌더예산 1MB: 두 관심사가 다르다 — 노드 폭주 vs parse/파싱 비용.)
검증은 "임계 함수가 parse 전에 올바로 판정한다"를 단언(B1–B4 경계·초과)해 *순서 자체*를 핀으로 박았다.

## 사실 2 — "타입 받아 렌더러 고른다"의 정직한 생산자가 없으면, 메타 파이프라인 말고 렌더시점 휴리스틱

원 보고: "응답이 markdown인데 평문으로 그려진다. JSON이면 어쩌지? **이런 타입 정보를 받을 수 있다면**
렌더러도 바꿀 수 있겠다." 직관은 "스트림/스키마/모델에 contentType을 실어 받자"였다. 실측하니
**LangGraph 런타임이 모델 출력에 포맷을 안 붙인다** — 스트림 프레임·`MessageOut`·`Message` 어디에도
타입 필드가 없고, 붙이려면 *추측값*을 스키마 표면까지 늘려 박아야 한다(정직한 생산자 부재).

그래서 타입 메타 파이프라인(stream→schema→model→영속)은 **비목표**로 두고, *같은 추측*을 **렌더시점
프런트 휴리스틱**으로 공짜로 했다(보조도구에 스키마 확장=과한 금칠, [[core-is-model-config-and-memory]]).
판별은 "전체 응답이 JSON *문서*(`{`/`[` 시작 + `JSON.parse` 성공 + object/array)"일 때만 — `42`/`true`는
유효 JSON이어도 markdown(거짓양성 차단).

곁들인 분별: **형식 추론은 스트림 완료에 게이트**한다. 실제 도구(ChatGPT·Claude.ai)도 "끝에서 형식을
추론"하지 않고 프로토콜이 *블록 경계에서 타입을 통보*한다(typed content blocks·input_json_delta·artifact
fence). 우리는 그 타입 프로토콜이 없으므로 차선책: `streaming===false`에서만 JSON 체크 1회, 스트리밍
중엔 *곱게 degrade하는* markdown 경로(부분 JSON이 트리로 깜빡이지 않음). 추가 상태 0 — 이미 계산하던
`streaming && isLast`를 그대로 prop으로 흘렸다.

F1(parse 프리즈, 위 사실1)·F2(깊은 중첩 재귀→`MAX_DEPTH` 중단)·F3(markdown 이미지 자동로드가
*추적 픽셀*로 admin IP/referrer 누출→`<img>`를 링크로 치환, 자동 fetch 0·클릭만 사용자 의도)·
F4(`JSON.parse` 16자리+ 정수 정밀도 손실→디버그 콘솔이 *틀린 ID* 표시→`/[0-9]{16,}/` 검출 시 원문
verbatim pre). 보조 표시 도구지만 넷 다 *싸서* 고쳤다 — "과한 금칠 금지"는 *비쌀 때* 적용이지, 1–3줄
covering 가드를 빼라는 뜻이 아니다. nit 2건(truncate해놓고 "원문 보존"이라 한 문구=정직성, src 없을 때
빈 href 링크)도 정정 — 보고는 정직해야 한다.

## 핵심 한 줄

가드는 *설치*가 아니라 *덮음*이고, 덮음은 범위뿐 아니라 **순서**다 — 비싼 연산을 막는 가드는 그 연산
*앞*에 있어야 하고(검사지점이 비용지점 뒤면 설치돼도 못 막음), 옮길 땐 *모든* 비싼 경로(parse·파싱·렌더)를
앞에서 덮어야 한다. 그리고 "타입을 받아 렌더러를 고르고 싶다"는 욕구는 *정직한 타입 생산자가 있을 때만*
파이프라인이고, 없으면 추측을 스키마로 늘려 박지 말고 렌더시점 휴리스틱+완료 게이트로 공짜로 해라.
