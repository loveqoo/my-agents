# Learning Index  (.dev/learning/)

> 각 줄: `NNN 제목 — 언제 열까(후크) [키워드]`. Context 단계에서 이 인덱스를 먼저 읽고,
> 후크로 불충분할 때만 해당 full 파일을 연다.

- 001 경로 기반 메모리의 한계와 폴더 리네임 — 폴더 이동·리네임으로 경로 키 메모리가 고아 되니 영속 맥락은 repo에 둘 때 [memory,rename,path]
- 002 User 스코프는 허락받고 접근, 프로젝트 내용으로 오염 금지 — User 스코프(경로 메모리)에 쓰려 할 때·cross-project 아닌 내용을 섞지 말 것 [memory,scope,user]
- 003 재촉하지 말 것 (사람 속도에 맞춰라) — 매 턴 끝을 승인·커밋·다음단계로 몰아가려는 충동이 들 때 [process,review,pacing]
- 004 AskUserQuestion에 CJK는 \u 이스케이프 말고 직접 작성 — 도구 입력 문자열에 한글 넣을 때 JSON 파싱 실패가 나면 [askuser,cjk,json]
- 005 docs 스펙은 AI 초안 + 인간 검토 — docs/spec을 누가 쓰나 헷갈릴 때(AI 초안→인간 검토) [docs,spec,review]
- 006 Qwen(MLX) thinking 비활성으로 콘텐츠 누락 방지 — 로컬 MLX/Qwen 스트림이 빈 응답·추론토큰 폭주일 때 [mlx,qwen,thinking]
- 007 백그라운드 서버 재기동 시 포트 점유를 확실히 정리 — 서버 재기동이 address-in-use거나 옛 코드가 응답할 때 [server,port,restart]
- 008 handoff 디자인을 진짜 antd로 이식하는 플레이북 + antd-x v2 함정 — design handoff 번들을 antd로 이식하거나 antd-x 함정에 막힐 때 [antd,handoff,design]
- 009 Mem0를 로컬 MLX로 구성하기 (LLM·임베딩·벡터스토어) — mem0를 외부 OpenAI 없이 전부 로컬 MLX로 구성할 때 [mem0,mlx,embedding]
- 010 create_react_agent + 회상 주입은 단일 system 프롬프트로 — 회상·동적지시를 별도 system 메시지로 넣어 다중 system 에러 날 때 [langgraph,system,mem0]
- 011 Alembic를 async FastAPI 시작 시 안전하게 돌리기 — async lifespan에서 alembic upgrade가 running-loop로 깨질 때 [alembic,async,fastapi]
- 012 런타임 설정은 단일 소스(레지스트리)에서, env는 부트스트랩만 — LLM/임베딩 설정을 어디서 읽을지·env 폴백을 둘지 정할 때 [config,registry,env]
- 013 매 응답 `[단계 N — 이름]` 표기를 빠뜨리지 말 것 — 자율 실행에 몰입해 단계 헤더를 빠뜨리고 있을 때 [process,header,loop]
- 014 비밀값 at-rest 암호화(Fernet)와 함정 — api_key·토큰을 DB에 암호화 저장하거나 복호화 폴백을 짤 때 [secret,fernet,crypto]
- 015 API Bearer 토큰 인증과 함정 — API에 Bearer 인증 붙이거나 토큰 fail-closed·gitignore 처리할 때 [auth,bearer,token]
- 016 테스트 가이드는 실제 UI를 확인하고 쓴다 — 사용자에게 화면 조작 가이드를 줄 때 코드 안 보고 추정 말 것 [ui,guide,verify]
- 017 자유 문자열을 하드코딩 맵으로 조회하면 렌더가 통째로 죽는다 — 백엔드 자유문자열을 프런트 상태맵으로 조회해 페이지가 백지될 때 [react,status,crash]
- 018 antd 6 어드민 모바일 반응형 플레이북 — antd handoff 이식물이 모바일에서 깨질 때 반응형·Sider 오버레이 패턴 [antd,mobile,responsive]
- 019 mem0 메모리 스코핑: user_id 축, thread_id 직교, entity-id 제약 — mem0 user_id와 LangGraph thread_id를 한 변수로 합치려 할 때 [mem0,scope,thread]
- 020 mem0 공유 백엔드(pgvector) + DSN은 분해 말고 위임 — mem0 벡터스토어를 N-인스턴스 공유·pgvector로 옮기거나 DSN 다룰 때 [mem0,pgvector,dsn]
- 021 mem0의 가치는 memory_type이 아니라 스코프·추출·엔티티그래프 — 부분구현만 보고 "mem0 이점 없음"이라 단정하려 할 때 [mem0,extraction,probe]
- 022 뷰포트 브레이크포인트는 옆 패널 폭을 모른다 — 고정폭 형제 패널이 있는데 반응형 분기를 뷰포트 폭으로 할 때 [responsive,breakpoint,layout]
- 023 프로세스 규칙은 가시적 강제 산출물이 있어야 지켜진다 — 새 절차 규칙을 만드는데 가시적 강제물 없이 텍스트만 남길 때 [process,enforcement,visibility]
- 024 적대적 픽스처로 "특정 콘텐츠 깨짐" 재현 + Playwright antd 오버레이 사이더 네비 함정 — 시드로 재현 안 되는 콘텐츠 깨짐·antd Tag 줄바꿈을 잡을 때 [adversarial,fixture,antd]
- 025 시드/목업이 레지스트리와 어긋나면: 데이터 마이그레이션 + 단일 상수 — 시드된 라이브 DB가 레지스트리와 drift 났을 때(소스만 고치면 안 됨) [seed,migration,drift]
- 026 Mock/스텁은 런타임 분기가 아니라 레지스트리 데이터 + 계약 구현으로 — mock/테스트더블을 if-mock 런타임 분기로 넣으려 할 때 [mock,registry,contract]
- 027 프런트 필터는 백엔드 계약의 가드가 아니다 — 프런트에서 입력 걸렀다고 서버 불변식이 안전하다 믿을 때 [frontend,backend,guard]
- 028 서버측 외부 URL fetch는 "보조도구"여도 보안 표면이다 — 사용자가 준 URL을 서버가 fetch하는 기능 만들 때(크기·타임아웃·SSRF) [ssrf,fetch,security]
- 029 same-origin 프록시는 cross-origin 문제 "클래스"를 통째로 없앤다 — CORS·mixed-content·cert를 env로 땜질하며 원격 fetch 회귀가 날 때 [cors,proxy,origin]
- 030 UI 변경은 진짜 브라우저로 직접 확인한다 (사람 스샷 의존 X) — UI 검증을 사용자 스샷에 의존해 루프가 느려질 때 [ui,playwright,verify]
- 031 의도적 쓰기 채널은 스코프-태깅이 닫은 누출을 되연다 — 에이전트 자가기록 도구를 열어 PII 누출이 재유입될 때(도구 프롬프트≠격리) [mem0,scope,leak]
- 032 antd의 포털 렌더·전체 마운트가 단순 브라우저 셀렉터를 깨뜨린다 — antd Select 옵션·비활성 Tabs 때문에 Playwright 셀렉터가 헛돌 때 [antd,playwright,selector]
- 033 alembic autogenerate는 metadata에 없는 "남의 테이블"을 drop하려 든다 — autogenerate가 mem0 등 외부 런타임 테이블을 drop_table로 끼울 때 [alembic,autogenerate,drop]
- 034 공유·실 DB 검증은 불변식 + 델타로 (절대수치 금지) — 기존 데이터 많은 실 DB를 절대수치로 단언해 검증이 깨질 때 [verification,invariant,delta]
- 035 가드는 진실원을 검사하라, 사본 말고 (+ 초록 verify ≠ 견고) — 차원·불변식 가드를 사본/메타데이터로 검사하거나 초록 verify를 믿을 때 [guard,source,adversarial]
- 036 이상 신호가 인프라를 가리켜도, 의존성보다 내 측정을 먼저 의심하라 — 불일치가 화려한 인프라 버그를 가리킬 때 의존성 만지기 전 계측 확인 [measurement,probe,regex]
- 037 파괴적 노브엔 바닥을 깔아라 (경계값이 곧 delete-all) — retention=0 같은 "없음처럼 보이는 경계값"이 최대 파괴로 매핑될 때 [adversarial,destructive,boundary]
- 038 적대 리뷰는 *네가 상상한 실패*가 아니라 *상상 못 한 범주*를 찾는다 — 파괴적 작업에 자가 불변식+자가 테스트만 깔고 출하하려 할 때 [adversarial,invariant,review]
- 039 "drop-in 가능"은 두 번째 구현을 출하해야 *측정*된다 — 백엔드 추상화가 한 구현에 안 묶였다고 주장만 할 때(Protocol 누수) [abstraction,protocol,dropin]
- 040 실인프라 통합 테스트는 *단위 게이트와 적대 리뷰가 둘 다 못 보는* 결함을 잡는다 — 게이트를 단위+적대로만 검증하고 실 DB·요청간 글루를 안 태울 때 [integration,gate,drift]
- 041 "리밋"은 *원천 바이트*를 막아야 한다 — 선버퍼링·프레임 카운트는 캡이 아니다 — .content/.aiter_lines 위에서 사이즈 캡 걸어 OOM이 안 막힐 때 [limit,bytes,ssrf]
- 042 이동·리네임은 *양방향*으로 참조를 깬다 — 한 방향만 고치면 green이지만 깨져 있다 — 파일 옮기고 들어오는 참조만 갱신할 때(나가는 상대링크·단방향 검증 함정) [move,refs,relative-path,verify]
- 043 "표시가 거짓"인 결함은 코드층·데이터층으로 갈라 고친다 — 화면 수치/상태가 실제와 어긋날 때(코드가 잘못 읽음 vs 데이터가 가짜, 출하 리듬이 다름) [honesty,code-vs-data,batch-split]
- 044 "안전장치를 깔았다"는 "안전하다"의 증거가 아니다 — 적용 범위 경계를 적대자가 짚는다 — 서버측 fetch/probe에 가드 깔고 follow_redirects·재해석을 안 끌 때(가드는 최초URL만, 부수효과는 체인끝) [ssrf,guard,boundary,toctou,redirect]
- 045 통합 픽스처가 데모 시드에 결합하면, 카탈로그를 비울 때 통합 rung이 함께 증발한다 — 시드 데모 데이터(에이전트/MCP/권한)를 지울 때 그걸 픽스처로 빌려쓰던 통합 테스트·프로브·브라우저샷도 같이 죽음(040의 반대편: rung을 잃지 않는 법, self-fixture로 격리) [integration,fixture,seed,self-fixture,rung]
- 046 per-operation 타임아웃은 전체 작업의 deadline이 아니다 — 서버측 스트림 fetch에 httpx timeout(=per-read)만 걸고 "타임아웃 방어"라 주장할 때; slow-trickle이 코루틴 무한홀드 → asyncio.timeout(전체)로 감싸야 참. 041(WHERE 바이트)·044(WHICH URL)와 같은 메타패턴 "서브유닛 방어는 전체를 못 묶는다"의 WHICH-span 축 [timeout,stream,deadline,ssrf,slowloris]
- 047 성공-카운터에 건 멱등성은 부분 실패를 "완료"로 오인한다 — doc_count>0 같은 성공분만 세는 캐시로 멱등 스킵 걸 때(부분 적재 후 재실행이 누락분 안 채움); 완료 단위 집합으로 판단해야. 041/044/046 메타패턴의 WHICH-set(완전성) 축 [idempotency,counter,self-heal,partial,adversarial]
- 048 한 수명 경계(생성)에 건 보장은 수명 전체를 못 덮는다 — 다른 서브시스템이 그 사이 깬다 — 불변식을 생성 시점에만 보장하고 "확보"라 선언할 때(배치/GC/cascade가 후반에 그 행을 지움); 보장은 생성→소멸 전 구간·모든 쓰기주체에 걸쳐 표현하고 검증을 수명 끝까지+대조군으로. 041(WHERE)·044(WHICH)·046(WHICH-span)·047(WHICH-set) 메타패턴의 WHEN(수명) 축 [lifecycle,guard,subsystem,hil,cleanup,adversarial]
