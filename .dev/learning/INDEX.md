- 028 서버측 외부 URL fetch는 "보조도구"여도 보안 표면이다 — 사용자가 준 URL을 서버가 fetch하는 기능 만들 때(크기·타임아웃·SSRF) [ssrf,fetch,security]
- 029 same-origin 프록시는 cross-origin 문제 "클래스"를 통째로 없앤다 — CORS·mixed-content·cert를 env로 땜질하며 원격 fetch 회귀가 날 때 [cors,proxy,origin]
- 030 UI 변경은 진짜 브라우저로 직접 확인한다 (사람 스샷 의존 X) — UI 검증을 사용자 스샷에 의존해 루프가 느려질 때 [ui,playwright,verify]
- 031 의도적 쓰기 채널은 스코프-태깅이 닫은 누출을 되연다 — 에이전트 자가기록 도구를 열어 PII 누출이 재유입될 때(도구 프롬프트≠격리) [mem0,scope,leak]
- 032 antd의 포털 렌더·전체 마운트가 단순 브라우저 셀렉터를 깨뜨린다 — antd Select 옵션·비활성 Tabs 때문에 Playwright 셀렉터가 헛돌 때 [antd,playwright,selector]
- 033 alembic autogenerate는 metadata에 없는 "남의 테이블"을 drop하려 든다 — autogenerate가 mem0 등 외부 런타임 테이블을 drop_table로 끼울 때 [alembic,autogenerate,drop]
- 034 공유·실 DB 검증은 불변식 + 델타로 (절대수치 금지) — 기존 데이터 많은 실 DB를 절대수치로 단언해 검증이 깨질 때 [verification,invariant,delta]
- 035 가드는 진실원을 검사하라, 사본 말고 (+ 초록 verify ≠ 견고) — 차원·불변식 가드를 사본/메타데이터로 검사하거나 초록 verify를 믿을 때 [guard,source,adversarial]
- 036 이상 신호가 인프라를 가리켜도, 의존성보다 내 측정을 먼저 의심하라 — 불일치가 화려한 인프라 버그를 가리킬 때 의존성 만지기 전 계측 확인 [measurement,probe,regex]
- 037 파괴적 노브엔 바닥을 깔아라 (경계값이 곧 delete-all) — retention=0 같은 "없음처럼 보이는 경계값"이 최대 파괴로 매핑될 때 [adversarial,destructive,boundary]
- 038 적대 리뷰는 *네가 상상한 실패*가 아니라 *상상 못 한 범주*를 찾는다 — 파괴적 작업에 자가 불변식+자가 테스트만 깔고 출하하려 할 때 [adversarial,invariant,review]
- 039 "drop-in 가능"은 두 번째 구현을 출하해야 *측정*된다 — 백엔드 추상화가 한 구현에 안 묶였다고 주장만 할 때(Protocol 누수) [abstraction,protocol,dropin]
- 040 실인프라 통합 테스트는 *단위 게이트와 적대 리뷰가 둘 다 못 보는* 결함을 잡는다 — 게이트를 단위+적대로만 검증하고 실 DB·요청간 글루를 안 태울 때 [integration,gate,drift]
- 041 "리밋"은 *원천 바이트*를 막아야 한다 — 선버퍼링·프레임 카운트는 캡이 아니다 — .content/.aiter_lines 위에서 사이즈 캡 걸어 OOM이 안 막힐 때 [limit,bytes,ssrf]
- 042 이동·리네임은 *양방향*으로 참조를 깬다 — 한 방향만 고치면 green이지만 깨져 있다 — 파일 옮기고 들어오는 참조만 갱신할 때(나가는 상대링크·단방향 검증 함정) [move,refs,relative-path,verify]
- 043 "표시가 거짓"인 결함은 코드층·데이터층으로 갈라 고친다 — 화면 수치/상태가 실제와 어긋날 때(코드가 잘못 읽음 vs 데이터가 가짜, 출하 리듬이 다름) [honesty,code-vs-data,batch-split]
- 044 "안전장치를 깔았다"는 "안전하다"의 증거가 아니다 — 적용 범위 경계를 적대자가 짚는다 — 서버측 fetch/probe에 가드 깔고 follow_redirects·재해석을 안 끌 때(가드는 최초URL만, 부수효과는 체인끝) [ssrf,guard,boundary,toctou,redirect]
- 045 통합 픽스처가 데모 시드에 결합하면, 카탈로그를 비울 때 통합 rung이 함께 증발한다 — 시드 데모 데이터를 지울 때 그걸 픽스처로 빌려쓰던 통합 테스트·프로브·브라우저샷도 같이 죽을 때(040의 반대편, self-fixture로 격리) [integration,fixture,seed,self-fixture,rung]
- 046 per-operation 타임아웃은 전체 작업의 deadline이 아니다 — httpx per-read timeout만 걸고 "타임아웃 방어"라 주장할 때(slow-trickle이 코루틴 무한홀드→asyncio.timeout 전체로 감싸야); 041(WHERE)·044(WHICH)의 WHICH-span 축 [timeout,stream,deadline,ssrf,slowloris]
- 047 성공-카운터에 건 멱등성은 부분 실패를 "완료"로 오인한다 — doc_count>0 같은 성공분만 세는 캐시로 멱등 스킵 걸 때(부분 적재 후 재실행이 누락분 안 채움); 완료 단위 집합으로 판단할 것 [idempotency,counter,self-heal,partial,adversarial]
- 048 한 수명 경계(생성)에 건 보장은 수명 전체를 못 덮는다 — 불변식을 생성 시점에만 보장하고 "확보"라 선언할 때(배치/GC/cascade가 후반에 그 행을 지움); 보장은 생성→소멸 전 구간·모든 쓰기주체에 걸쳐 표현하고 검증을 수명 끝까지 [lifecycle,guard,subsystem,hil,cleanup,adversarial]
- 049 상태를 지우는 청소는 재생성 소스가 살아있으면 무의미하다 — 행이 아니라 발생원을 고쳐라 — 파괴적 정리 후 정크가 재발할 때(부팅시드·크론·다른 프로세스 env·리로더가 DB 밖에서 재생성); 삭제=증상·발생원=원인 [regeneration,source-vs-state,seed,env,cleanup,whence]
- 050 파괴 헬퍼의 한 연산(delete)에 건 가드는 권한 등가인 형제 연산(create)을 안 덮는다 — 보호대상에 대한 모든 동사를 위협모델에 넣을 때(delete막고 create로 super승격 열어두면 같은등급 피해); 가드는 한 술어로 모든 입구에 [guard,operation-symmetry,escalation,privilege,idempotency,adversarial]
- 051 실 클라이언트 제약을 무시하는 검증 도구는 거짓 green을 준다 — curl은 브라우저가 아니다 — 인증/쿠키/origin 경로를 curl로 green 받고 안심할 때(curl은 Secure/SameSite/CORS 강제 안 함); 인증경로는 Playwright로 한번 태울 것 [verification,tooling-fidelity,cookie,secure,browser,false-green]
- 052 누출은 비신뢰 쓰기 채널을 잘라 막는다 — 데이터도 기능전체도 아니다 — 한 저장소에 신뢰(인간게이트)·비신뢰(LLM자가기록) 채널이 같이 쓸 때; 수리단위는 채널이고 절단면은 "어느 쓰기주체가 신뢰불가"로 그어짐 [leak,write-channel,human-gate,trust-boundary,remove-tool,regression-guard]
- 053 신원 보강은 스코프 엔드포인트로, 고권한 엔드포인트 재사용 금지 — 저권한 화면을 다른 곳 신원(email·소유자)으로 보강할 때 고권한 엔드포인트(users:manage) 재사용하면 권한등급이 조용히 끌어올려짐; 최소권한 read 엔드포인트 신설 [authz,coupling,scoped-endpoint,least-privilege,response-model,leak,adversarial]
- 054 인가는 모든 직교 축에서, 강제는 서버에서 — 공유저장소 행에 주체가 대상의 행으로 접근하는데 가드를 한 축만 걸 때(/{user_id}/{mem_id}면 축도 둘, 주체가드만이면 path위조로 남의 행 삭제); per-route 독립 재판정+fail-closed [authz,orthogonal-axis,path-spoof,defense-in-depth,fail-closed,server-enforce]
- 055 새 아웃바운드 경로는 네트워크 가드를 자동 상속하지 않는다(의존성은 위험 기본값을 들고 옴) — 새 외부호출 경로(새 클라/통합)를 추가할 때("SSRF가드 있다"≠"이 경로에 적용", follow_redirects=True 기본이 사설/메타데이터 재유도); 경로마다 가드 재확인·새 의존성 fail-closed 잠금 [ssrf,redirect,outbound-path,dependency-default,follow-redirects,fail-closed,guard-coverage]
- 056 낙관적 비동기 상태: "도착"은 "유효"가 아니다 — set엔 롤백, 도착물엔 단조 토큰 — 비동기로 데이터 불러와 화면상태 바꾸는데 대기중 다른선택/요청실패 가능할 때(stale도착물이 최신선택 덮음·실패후 미롤백); 단조토큰 seq+catch롤백, 타자 적대 필수 [async,optimistic-update,staleness-token,rollback,race,adversarial,false-green]
