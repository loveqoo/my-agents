# 107 — 성장하는 카탈로그를 폼에 넣을 땐: 모드 조건부 + 종류별 접이식 + 카운트 배지

## 맥락

능력 피커(스펙 106)를 종류별 체크박스 벌판으로 냈더니 사용자가 화면 보고 "RAG 늘면 폼이 계속
복잡해지고 어렵다" 지적. 컬렉션/MCP/에이전트는 **개수가 무한정 늘 수 있는 카탈로그**인데 이를 flat
체크박스로 나열하면 폼 높이가 카탈로그 크기에 비례해 커진다.

## 배운 것

### flat 목록은 "성장하는 카탈로그"에 안 맞는다 — 세 겹으로 접어라

폼에 개수가 늘 수 있는 항목을 넣을 때, 폼 높이를 카탈로그 크기와 **분리**하는 세 장치:

1. **모드 조건부 표시** — 그 항목이 특정 모드에서만 의미 있으면, 그 모드일 때만 렌더. 능력은
   오케스트레이터 impl에서만 쓰이니 오케스트레이터일 때만 노출 → 대다수(일반 에이전트) 화면에서
   **아예 사라짐**. "안 쓰는 사람에겐 0 픽셀"이 가장 강한 단순화.
2. **종류별 접이식(accordion), 기본 접힘** — 종류(kind)마다 패널로 묶고 접어 둔다. 펼친 종류만 목록
   노출 → 종류가 4개여도, 각 종류에 100개여도 기본은 헤더 4줄. 선택 있는 패널만 기본 펼침(편집 시
   켜진 것 바로 보이게).
3. **접힌 헤더에 카운트 배지** — `선택수/전체수`(`1/2`·`0/12`)를 헤더에 얹으면 **펼치지 않아도**
   무엇이 얼마나 켜졌는지 한눈에. 접힘이 정보를 숨기지 않게 하는 핵심 — 배지 없는 accordion은
   매번 펼쳐 확인해야 해 오히려 불편.

핵심은 **폼 높이 ⟂ 카탈로그 크기**. flat은 O(항목수) 높이, 이 셋은 O(종류수) 높이(+펼친 것만).

### 부수: 데이터 모델 안 건드리고 표현만 교체

선택 상태(`form.capabilities`)·토글(`toggleCap`)·저장 배선은 그대로 두고 **렌더링 계층만** flat→
accordion으로 교체했다. UX 리워크가 저장 경로를 안 건드리니 회귀 위험이 표현 계층에 갇힌다.

## 함정

- **antd Checkbox를 Playwright로 토글**: 라벨 텍스트 span 클릭은 불안정 → input `.check()` 쓸 것
  (`label.ant-checkbox-wrapper … locator('input').check()`).
- **vite IPv6-only**(learning 106) — 브라우저 검증은 `localhost`(=`::1`).

## 적용

관리 폼에 "늘 수 있는 목록"(태그·컬렉션·도구·멤버 등)을 넣을 땐 flat 체크박스 대신: 모드 조건부 →
종류별 접이식 → 카운트 배지. 사용자 UX 피드백은 **AskUserQuestion 미리보기(ASCII 목업)로 방식을
좁혀** 추측 없이 즉결. 관련: [[106-config-roundtrip-completeness-and-namespace-as-authoring-layer]],
[[verify-ui-in-browser-proactively]], [[explain-plainly-not-one-sided]].
