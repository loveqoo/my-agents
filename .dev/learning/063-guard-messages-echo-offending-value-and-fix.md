# 063 — fail-closed 가드 메시지는 *위반한 값 + 그 값을 넣은 조치 예시*를 담아라(노브 이름만으로는 부족)

## 언제 떠올리나
fail-closed 가드(SSRF·인증·검증·프리플라이트)가 요청을 막고 4xx/부팅중단을 낼 때, 그 에러
메시지를 쓸 때. 특히 "이 노브로 허용 가능"처럼 *기능*만 가리키고 끝내고 싶을 때.

## 명제
가드가 막는 순간 운영자가 알아야 할 건 셋이다: (1) **무엇이** 막혔나(위반한 구체값), (2) **왜**,
(3) **어떻게** 푸나(정확한 설정 키 + *그 값을 이미 넣은* 예시). 노브 이름만 주면("A2A_ALLOWED_HOSTS로
허용 가능") 운영자는 *무엇을* 그 노브에 넣어야 하는지 다시 추론해야 한다 — 가드는 이미 그 답
(차단된 host)을 손에 쥐고 있으면서 안 알려준 것이다. 메시지에 차단값을 echo하고 `KEY=<그 값>`
형태의 복붙용 예시를 넣으면, 추론 단계가 사라지고 메시지 자체가 처방이 된다.

## 표본 (A2A_ALLOWED_HOSTS, net_guard.py)
이전: `"사설/내부 대역으로의 요청은 차단됩니다(A2A_ALLOWED_HOSTS로 허용 가능)"` — 노브만 가리킴.
이후: `f"...차단됩니다(host={host}). 개발용 mock 등 의도된 대상이면 환경변수 A2A_ALLOWED_HOSTS에
이 호스트를 추가하세요(쉼표구분, 예: A2A_ALLOWED_HOSTS={host}). 변경 후 API 재기동 필요."`
가드는 `host`를 이미 resolve 단계에서 알고 있었다 — 그걸 메시지에 흘려 `예: A2A_ALLOWED_HOSTS=127.0.0.1`
까지 만들어주면 dev가 mock을 띄울 때 한 줄을 그대로 복사한다. `.env.example`에도 같은 키를
주석으로 두어(기본 차단·쉼표구분·재기동 필요 설명) "에러를 만나기 전에" 발견 가능하게 했다.

## 처방
1. **막은 값을 echo.** 가드가 검사 중 이미 가진 구체값(host·dim·user_id·DSN 마스킹본)을 메시지에 넣어라.
2. **노브가 아니라 처방을 줘라.** `KEY로 허용 가능`이 아니라 `KEY=<그 값>` 복붙 예시 + 적용 조건
   (재기동 필요? 어디 파일?)까지. 운영자의 다음 행동을 메시지가 대신 작성해준다.
3. **막기 전에 발견 가능하게.** 같은 노브를 `.env.example`/README에 주석으로 둬 "에러를 만나야만
   존재를 아는" 설정을 없앤다. 에러 메시지(사후)와 예제(사전)는 같은 키를 가리켜야 한다.
4. **민감값은 마스킹하되 형태는 보여라.** 비번/토큰은 `***`로 가리되 *무엇을* 고치는지는 남긴다
   (G1 프리플라이트의 마스킹 DSN — 스펙 058).

## 가족 관계
- **061 graceful-degradation은 원자성을 못 넘는다**(스펙 058 G1): 그 결론이 "정직한 fail-closed +
  또렷한 조치메시지"였다. 이 learning은 그 "또렷한 조치메시지"를 *한 단계 더* — 값 echo + 복붙
  예시 — 로 조작화한다.
- **044/055 가드 적용범위**: 거기는 가드가 *덮는 범위*, 여기는 가드가 *막을 때 말하는 법*. 같은
  가드의 커버리지 축 vs 관측가능성(observability) 축.
