# 002 — User 스코프는 허락받고 접근, 프로젝트 내용으로 오염 금지

날짜: 2026-06-19

## 무엇이 잘못됐나
- 2단계(Context)에서 사용자가 User 스코프 기록 *위치*로 "경로 기반 메모리"를 골랐다.
- 나는 이를 "User 스코프 메모리에 언제든 써도 됨"으로 확대 해석했다.
- 게다가 `my-agents`의 6단계 루프·주력 저장소 같은 **프로젝트 한정 내용을 cross-project여야 할 User 스코프에 섞어** 넣었다.
- 사용자 지적: "User 스코프는 내가 허락할 때만 접근해. 이 프로젝트 내용으로 User 스코프를 더럽히지 말라."

## 그래서 우리의 규칙
- **User 스코프(경로 기반 메모리 등)는 사용자가 명시적으로 허락할 때만 읽고/쓴다.** 위치 선택 ≠ 상시 쓰기 허가.
- **User 스코프에는 프로젝트 한정 내용을 넣지 않는다.** User 스코프는 cross-project 성격(모든 프로젝트에 적용되는 사용자 정체성·선호)만 담는다.
- 프로젝트 한정 맥락은 저장소 안에 둔다 → `CLAUDE.md`, `.dev/`, `docs/`. (cf. [[001]] 경로 기반 메모리의 한계)

## 적용
- User 스코프에 쓸 일이 생기면 먼저 "여기에 적어도 될까?"를 묻고, 적을 내용이 정말 cross-project인지 검증한다.
- 의심되면 저장소(repo)에 적는 쪽을 기본값으로 한다.
