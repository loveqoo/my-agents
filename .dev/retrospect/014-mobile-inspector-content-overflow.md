# 014 — 모바일 인스펙터 콘텐츠 오버플로 (AI 회고)

작업: spec 022 / 플레이그라운드 인스펙터 모바일 깨짐 + 전 페이지 회귀 검증.

## 무엇이 잘 됐나

1. **단정을 미뤘다.** 시드 데이터로는 오버플로가 재현되지 않았다. 여기서 "재현 안 됨,
   문제 없음"으로 닫을 수 있었지만, 사용자 보고("특정 콘텐츠가 깨짐")와 내 측정이 어긋난다는
   사실 자체를 신호로 봤다(memory `probe-deeper-before-concluding`). 측정을 의심 →
   적대적 콘텐츠 픽스처로 잠복 버그를 강제 표출 → 7개 오프endер를 결정적으로 잡았다.
   회상한 memory가 실제로 행동을 바꾼 케이스.

2. **Context 단계에서 자산을 실제로 적용했다.** learning 018(모바일 오버플로 근본 원인)과
   017(shared 컴포넌트엔 줄바꿈 보호 적용)을 grep·읽고, "playground 인스펙터는 017 당시
   별도 경로라 누락됐다"는 가설을 세워 바로 그 7곳을 찾았다. 회고가 복리로 작동한 증거.

3. **검증을 타자에 맡겼고 영구 자산으로 남겼다.** codex GATE PASS(P1 없음), P2(메모리 태그
   break 규칙)·일관성 권고(overflowWrap > break-all) 둘 다 반영. 일회성 스윕에 그치지 않고
   `mobile-overflow.spec.ts` + config `mobile` 프로젝트로 회귀 가드를 박았다.

## 무엇이 아팠나 / 다음에 줄일 것

1. **Playwright 모바일 네비에서 시간을 흘렸다.** `getByRole('menuitem')`이 숨은 중복 항목을
   잡아 force 클릭마저 타임아웃 → 한참 헤맸다. → [[024]]에 `:visible` 스코프 패턴으로 박제.
   antd 오버레이 사이더를 Playwright로 칠 땐 첫 시도부터 이 패턴을 쓴다.

2. **FAIL 기준을 처음에 잘못 잡았다.** 1차 스윕이 antd Tabs nav-list와 빈 div를 오탐으로
   FAIL 처리. 페이지 레벨 가로 스크롤(docOF/contentOF)이 진짜 신호이고, antd 내부 클리핑
   요소는 정보성으로 강등해야 한다는 걸 뒤늦게 정리했다. → [[024]] 배운 것 3.

3. **시드 데이터의 한계를 더 일찍 의심했어야.** "짧은 콘텐츠는 잠복"은 일반 원리인데,
   재현 실패 직후가 아니라 한 박자 늦게 픽스처로 갔다. 다음엔 "줄바꿈/오버플로 버그는
   합성 극단 입력으로 먼저 친다"를 기본 절차로.
