# 062 — 미활성 초안 배지(이슈1) 회고

## 무엇을 했나
사용자 보고 "에이전트 수정 후 플레이그라운드에서 조회하면 바뀌지 않음"(이슈1)을, 사용자 결정
"미활성 배지로 안내 강화"에 따라 Playground에 미반영 초안 배지를 추가해 닫았다(스펙 078).

## 핵심 판단 — "버그"가 아니라 "안내 미도달"
Context에서 배선을 실측하니 증상은 설계대로였다: `update_agent`는 draft 버전에만 저장하고,
`list_agents`/서빙은 활성 `Agent.config`만 본다. 그리고 편집 화면(AgentsView)은 *이미* 충분히
안내하고 있었다(부제·토스트·Alert·`+초안` Tag). 여기서 멈추고 "정상 동작"이라 답할 수도 있었지만,
한 겹 더 들어가니 진짜 갭은 **증상이 나타나는 화면(Playground)에 신호가 0**이라는 점이었다.
사용자는 편집 직후 *다른 화면*으로 이동해 "안 바뀜"을 마주치는데, 거기엔 답이 없었다.

→ 그래서 백엔드 0줄, 표시층만 추가. 진실원(`versions.some(draft)`)으로 구동했고, 이미 직렬화돼
오는 versions 덕에 추가 API가 필요 없었다.

## 검증
- tsc 무에러(단위).
- 브라우저 양성·음성 둘 다(통합): 신규(draft) 에이전트 헤더 배지 1·드롭다운 초안 Tag 1,
  활성 에이전트 배지 0. 거짓 배지 배제를 음성 케이스로 못박음. 테스트 에이전트는 끝에서 삭제(상태 정리).

## 잘된 것 / 다음에
- 잘됨: "설계대로"에서 멈추지 않고 "그럼 왜 헷갈렸나"까지 판 것 → 자산화(learning 081).
- 잘됨: 신규 에이전트가 자연히 draft 상태(`create_agent` v1=draft)임을 이용해 시드 오염 없이
  양성 케이스를 만들고, 끝에서 삭제로 정리.
- 다음: 스테이징 상태를 도입할 땐 처음부터 "모든 소비 표면 열거"를 체크리스트로(learning 081).
