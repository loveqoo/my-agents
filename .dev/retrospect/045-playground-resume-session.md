# 045 — Playground 세션 이어가기: 낙관적 고정과 비동기 쓰기 타이밍 회고

스펙: docs/spec/055-playground-resume-session.md

## 무엇을 만들었나
사용자 보고: Playground에서 위험 도구(`delete_record`) 승인하러 다른 뷰로 갔다 오면
대화·세션이 초기화된다(React 메모리에만 있어 언마운트 소실). 요구는 직역 — **과거 세션을
골라 이어서 채팅**. DebugChat 헤더에 에이전트 피커 옆 세션 피커를 두고(수동 선택만),
`GET /sessions`에 `agent_id` 필터를 더해 활성 에이전트 세션만 추리고, 선택 시
`getSessionMessages`로 대화를 복원해 같은 session_id로 이어 보낸다.

## 잘 된 것
- **백엔드는 이미 영속·재개를 지원했다 — 빠진 건 "불러오는 UI"였다.** Context 단계에서
  chat.py(`resume_approval`·`_load_context`)와 049 lazy-create를 먼저 읽어 "새 기능"이 아니라
  "조회 배선"임을 확정. 진단이 범위를 작게 못박았다(필터 1개 + 피커 1개).
- **검증 사다리 3단을 다 태웠다.** 통합(verify_055, agent 필터 누출 0·counts 전역·preview
  절단 22단언), 브라우저 E2E(RESUME055_OK), 타자(codex) 적대 리뷰. 세 단이 잡는 결함이
  겹치지 않았다 — 통합은 시맨틱, E2E는 글루, 타자는 동시성.

## 아팠던 것 (그래서 배운 것)
### 1. 비동기 쓰기가 끝나기 전 언마운트를 트리거하면 쓰기가 중단된다 (E2E)
E2E STEP3가 처음엔 복원 실패 + 0턴 세션이 떴다. 원인: 전송 후 3초만에 뷰를 떠났는데
**스트리밍이 아직 진행 중**(스크린샷에 stop 버튼)이라 턴이 abort → 백엔드 `_persist`가
일어나기 전에 세션이 안 박혔다. 피커엔 리마운트로 생긴 다른 0턴 세션이 떴다. → **언마운트로
중단될 비동기 쓰기는, 그 쓰기가 끝났다는 신호를 기다린 뒤 언마운트해야** 한다. 9초 대기 +
헤더 칩에서 createdSid를 정확히 캡처(전체 body 정규식은 시드 세션 id에 오염됨)로 해결.
이건 테스트만의 문제가 아니라 **실제 사용자도** 답변 스트리밍 중 승인하러 떠나면 같은
중단을 겪는다 — 라이브 재연결(041 §7 빚)이 메우는 진짜 틈을 E2E가 우연히 재현한 것.

### 2. 낙관적 상태 고정엔 롤백 경로가 필요하다 (codex)
codex 1차가 잡은 HIGH 레이스 외에, 2차 재리뷰가 **새 버그**를 짚었다: session_id를 fetch
*전에* 먼저 고정(로드 완료 전 전송해도 올바른 세션)하는 낙관적 패턴인데, **fetch 실패 시
이전 session_id로 되돌리지 않아** 실패한 세션에 묶인 채 남았다. 낙관적 set은 catch에서
prevSid 롤백을 짝으로 가져야 한다. happy-path 자가 테스트는 이걸 절대 못 본다 — fetch가
성공하니까.

### 3. 비동기 도착물엔 단조 토큰이 필요하다 (codex)
에이전트 전환·연속 선택·리셋이 끼면 "오래된 fetch 도착물"이 최신 화면을 덮어쓴다.
`activeExtRef`(항상 최신 agentId) + `sessionLoadSeqRef`(단조 토큰)으로 도착시점 일치할 때만
반영, `resetConversation`도 시퀀스를 무효화. **"성공 콜백이 도착했다"≠"이 결과가 아직
유효하다"** — 비동기 UI의 기본 함정.

## 사용자 피드백이 바꾼 것
"세션 해시코드만 보여주면 너무 불편합니다." → preview(첫 사용자 메시지 80자 절단) 라벨을
주(主)로, 해시는 보조 메타로 강등. **식별 가능성은 UX의 핵심이지 장식이 아니다** — 해시는
기계의 키일 뿐 사람의 라벨이 아니다. 백엔드 1쿼리(`role='user'` ORDER BY created_at, 세션별
최초)로 N+1 없이 채웠고, 통합 테스트로 절단·첫메시지만·빈세션 None을 단언.

## 다음에 적용할 것
- 비동기 쓰기를 중단시킬 언마운트/이탈은 **쓰기 완료 신호를 기다린 뒤** 트리거(E2E·실UX 공통).
- 낙관적 상태 set은 **반드시 catch 롤백을 짝으로**.
- 비동기 도착물은 **단조 토큰으로 stale 판정** — 도착≠유효.
- 식별 라벨은 **사람이 읽는 값을 주로**, 키는 보조로. → learning 056.
