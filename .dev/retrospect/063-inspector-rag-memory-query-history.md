# 063 — 인스펙터 RAG·메모리 조회 이력(이슈5) 회고

## 무엇을 했나
사용자 보고 "플레이그라운드 인스펙터에 tag 조회·메모리 조회 이력 안 보임"(이슈5)을, 사용자 결정
"RAG 문서검색 + 메모리 둘 다"에 따라 닫았다(스펙 079). 회상 *쿼리*와 RAG *연결 컬렉션*을
인스펙터에 드러내, 결과가 비어도 조회 행위가 보이게 했다.

## 핵심 판단 — 갭은 "결과 없음"이 아니라 "행위 비가시"
Context에서 배선을 실측하니 trace는 회상 *결과*(memories)와 RAG *호출 결과*(mcp)만 실었다.
메모리가 활성이라 매 턴 `memory.search`가 도는데도 **0건이면 "메모리 타입 …, 0건"만** 남고,
RAG는 도구를 안 부르면 *연결돼 있었는지조차* 안 보였다. 여기서 "정상 동작(결과가 없을 뿐)"이라
멈출 수도 있었지만, 한 겹 더 보니 진짜 갭은 **조회의 *행위*(쿼리·대상·시도)가 결과가 비면
사라진다**는 점이었다 → 자산화(learning 082).

→ 백엔드는 메모리 쿼리 에코 한 키만 추가(RAG의 ragCollections/ragUnresolved/hits는 *이미* trace에
실려 있었는데 Inspector가 안 그렸을 뿐). 즉 대부분 **표시층 누락**이었다.

## 검증
- tsc 무에러(단위).
- 브라우저 양성·음성(통합, RAG079_PASS): 양성(Research Assistant, mem0+docs_kb/product_titles)은
  회상 조회 "0건 회상"(0건이어도 쿼리 노출 ✅)·문서 검색(RAG) 섹션 컬렉션 태그 2+search_documents
  카드(3건·52ms). 음성(Doc Translator, code)은 두 섹션 0(무회귀). MCP는 rag 제외돼 중복 없음.
- 0건 회상인데 "3건 검색"이 공존 — 인메모리 회상과 RAG 문서검색이 **다른 경로**임을 화면이 정직히
  분리해 보였다(메모리=mem0 의미회상, RAG=벡터 문서검색).
