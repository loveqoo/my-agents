# 041 — 에이전트 메모리 누출: 데이터가 아니라 LLM 쓰기 채널을 잘랐다 (스펙 051)

메모리 테스트 중 로직 오류를 사용자가 보고: 채팅에서 *"내 이름은 구남이야. 잘 기억해"*라고 하자
이름이 **agent_id 스코프(에이전트 전용·교차사용자)**에 저장됐다. 이름은 user_id(사용자) 메모리에
들어가야 하는데 교차사용자 채널로 샜다.

## 진단

`mem0_memories`를 직접 떠보니 한 턴이 **두 군데** 쓰여 있었다: 정상 `user_id` 행(자동 add,
infer=True)과 **누출 `agent_id`-only 행**(`0a6a05d1…`, "사용자의 이름은 구남이다."). 후자는
인-챗 자가기록 도구 `save_agent_knowledge`(`runtime.build_agent_memory_tool`)가 만든 것.

핵심은 **회상 메커니즘**이었다(`mem0_backend.search`): 축별 단일필터 검색 후 합집합 →
**agent_id로 태깅된 건 어떤 user_id 검색에도 잡힌다**(구조적 교차사용자). 그래서 처음 내가 제안한
"agent_id 위에 user_id를 같이 태깅(agent×user)" 격리책은 **틀렸다** — bare agent_id-축 검색이
여전히 전 유저에게 그 행을 회상시킨다. 그 자리에서 그 옵션을 철회했다. 유저 개인사실의 **유일한
누출-없는 집**은 `user_id`(agent_id 없이)다.

## 결정의 전환 (이번 회고의 진짜 교훈)

처음엔 사용자와 "agent 전용 메모리 기능을 **통째로 제거**하고 페르소나로 보완"으로 합의했다.
그래서 어드민 CRUD·패널까지 제거하는 큰 범위로 spec을 쓰려던 참에 사용자가 **방향을 틀었다**:

> 삭제를 취소합니다. **어드민이 수정할 수 있게 하고, 유저가 채팅에서 넣을 수 없게** 합시다.
> 이는 **페르소나를 보호**하는 데도 도움.

이 한마디가 경계를 정확히 그었다. agent_id 메모리에는 **두 쓰기 채널**이 있었다 —
(1) 채팅 LLM 자가기록(신뢰 불가), (2) 어드민 저작(인간 게이트, 스펙 029). **누출은 (1)에만 살았다.**
그러니 고칠 대상은 *데이터 저장소*나 *기능 전체*가 아니라 **(1) 채널 하나**다. (1)을 잘라내고
(2)와 회상은 그대로 둔다. 이게 learning 030/031의 원칙("교차사용자 쓰기는 LLM 말고 인간 게이트")과
정확히 맞물린다 — 어드민 큐레이션이 바로 그 인간 게이트이므로 살리는 게 맞다.

덕분에 변경이 58줄 삭제·12줄 추가로 쪼그라들었다(통째 제거였다면 백엔드+프론트 수십 곳).

## 고침

- `runtime.build_agent_memory_tool` 함수 삭제(NOTE 주석으로 사유 남김).
- `chat.py` 일반 경로·resume 경로의 도구 주입 2곳 제거. `recall_scope`의 agent_id는 **유지**.
- 누출 행 `0a6a05d1…` DB에서 삭제(before/after 확인, 정상 user_id 이름 행은 보존).
- 패널 안내문 "에이전트가 스스로 기록한…" → "관리자가 저작한…(채팅 자가기록은 차단됨)" 정정.

## 검증 — 적대 타자가 잡은 것

자가검증을 피하고 서브에이전트로 적대 리뷰를 돌렸다. leak-closed/admin-intact/recall-intact는
모두 YES로 확인됐지만, **`tests/verify_029_agent_memory.py`가 삭제된 함수를 부르며 깨진다**는 걸
잡았다(소스만 고치고 검증자산을 안 고친 self-fixture 결함). 그 테스트의 `test_self_write_tool`을
뒤집어 **`test_no_self_write_tool`** — "도구가 *부재*함"을 단언하는 회귀 가드로 바꿨다(누군가
되살리면 빨강). 전 스위트 통과.

## 다음에 다르게

- **누출을 고칠 땐 "어느 쓰기 채널이 신뢰 불가인가"를 먼저 가른다.** 같은 데이터에 신뢰 채널과
  비신뢰 채널이 같이 붙어 있으면, 데이터를 지우거나 기능을 통째 없애기 전에 **비신뢰 채널만**
  잘라낸다. 범위가 작아지고 인간 게이트는 보존된다. → learning 052.
- **소스를 지우면 그 소스를 검증하던 자산도 같은 턴에 고친다**(self-fixture 전면화). 적대 리뷰가
  못 잡았으면 다음 실행에서 빨강으로 터졌을 것.
- LLM 도구 가드가 *프롬프트 문구*면(여기선 "유저 개인정보 금지"), 프롬프트를 더 조이지 말고
  **도구 자체를 제거**한다(learning 031 재확인 — 프롬프트 ≠ 격리).

관련: 스펙 051, learning 031(프롬프트≠격리), 030(인간 게이트), 052(이 회고의 추출),
[[installed-guard-isnt-covering-guard]](가드는 설정/부재를 단언), 045(self-fixture).
