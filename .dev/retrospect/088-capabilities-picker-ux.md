# 088 — 능력 피커 UX: 조건부 + 접이식 (스펙 107)

## 무엇을 / 왜

스펙 106 능력 피커를 사용자가 화면에서 보고 지적: "설정화면이 편의성을 많이 해친다. RAG가 늘면
폼이 계속 복잡해지고 어렵게 느껴진다." 종류별 체크박스 벌판이라 컬렉션/MCP/에이전트가 늘수록 폼이
세로로 무한정 길어지는 게 문제. **화면을 직접 보고 나온 피드백**이라 즉시 반영.

## 한 일 (사용자 확정 방향)

- **조건부 표시**: 능력 칸은 오케스트레이터 impl일 때만 렌더 → 보통 에이전트 생성 시 아예 안 보임.
- **종류별 접이식**(antd Collapse): 종류마다 패널, 기본 접힘, 헤더에 `선택/전체` 카운트 배지. 선택 있는
  패널만 기본 펼침(편집 시 켜진 것 바로 보임). 늘 수 있는 종류(RAG/MCP/agent)엔 패널 내 검색(>6개).
- 데이터 모델·저장 배선 무변경(`form.capabilities`·`toggleCap` 그대로) — 순수 표현 계층 교체.

## 잘된 것

- **사용자 피드백을 AskUserQuestion 미리보기로 좁혀 즉결.** 세 컨트롤 방식을 ASCII 목업으로 비교
  제시 → 사용자가 "종류별 접이식" 선택 → 그게 곧 승인. 추측 없이 한 번에 방향 확정.
- **접힌 패널의 카운트 배지가 상태를 보존**: 펼치지 않아도 `1/2`·`0/12`로 무엇이 얼마나 켜졌는지 한눈에.
  RAG 100개여도 헤더 한 줄 — 폼 높이 안정.
- 브라우저(실 Chrome) 7/7 + 실화면 확인. 조건부 등장·기본 접힘·카운트·펼침·토글 반영 전부.

## 배운 것 / 함정

- **antd Checkbox 토글은 라벨 텍스트 span 클릭이 불안정** — Playwright `getByText(label).click()`가
  토글 안 됨. 표준은 input `.check()`(`label.ant-checkbox-wrapper … locator('input').check()`). 검증
  스크립트에서 이걸로 교체하니 초록. (UI 결함 아님 — 실사용 클릭은 정상, 테스트 셀렉터 함정.)
- **성장하는 카탈로그를 폼에 넣을 땐 flat 목록 금물**(learning 107): 무한정 늘 수 있는 항목은
  ①모드 조건부로 관련 없을 때 숨기고 ②종류별 접이식+카운트로 접어, 카탈로그 크기와 폼 높이를 분리.

## 검증 안 함(스킵 사유)

- 편집 모드 default-펼침(선택 있는 패널)은 코드·타입체크로 검증(별도 브라우저 런 생략) — 로직 단순
  (`defaultActiveKey`=선택 있는 그룹). 필요 시 후속 확인.
- 백엔드/왕복은 106서 무변경 — UI만 교체, 기존 verify 유효.
