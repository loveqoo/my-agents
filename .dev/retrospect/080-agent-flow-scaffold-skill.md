# 080 — 그래프 빌더 폐기 → 에이전트 플로우 스캐폴드 스킬(코드젠)

> 스펙 099. learning 099. 참고: spec 085(CustomAgent 인터페이스·실 노드 타임라인)·089(conformance 3-state·
> 신뢰 레지스트리 isinstance 게이트)·[[agent-source-three-way-a2a-external]].

## 무엇을

제안 #4 "트리노드 그래프 빌더"(시각 편집기)를 **폐기**하고, 그 목적(새 에이전트 플로우를 만들어
재사용)을 **Claude Code 스킬 기반 코드젠**으로 대체. 스킬(`agent-flow`)이 `CustomAgent` Protocol에
적합한 LangGraph 플로우 모듈을 `agent/flows/`에 저작하고, `_bootstrap_builtins()`에 신뢰 등록 2줄을
얹고, 089/085 검증까지 잇는다. 데모 산출물 `route`(조건분기 라우터)로 파이프라인을 엔드투엔드 실증.

## 잘된 것

- **폐기 결정이 복잡도를 크게 줄임**: 시각 빌더는 그래프 직렬화/렌더 계층 + *런타임 문자열→클래스
  해석* 경로를 새로 세워야 했고, 그게 신뢰 레지스트리 불변식(085 U5: eval/import 경로 없음)과 정면
  충돌했다. 코드젠 스킬은 **이미 검증된 시임** 위에 얹혀 신규 런타임 표면이 거의 0.
- **확장점 위치가 보안 속성을 결정한다는 걸 명시**: 런타임 확장(동적 로딩) = 신뢰경계 위협 / 저작시점
  코드젠(커밋·리뷰) = 신뢰경계 보존. 스킬 방식이라 이 속성이 *자연히* 지켜졌다(learning 099).
- **수용 게이트를 발명하지 않음**: 생성물의 합격 기준을 새로 만들지 않고 기존 089(conformance)·085
  (구조·실 노드 타임라인)를 재사용 → "생성 → 자가 수용" 루프가 닫힘. 데모 flow가 스펙의 자체 실증.
- **조건분기로 과적합 재측정**: route는 plan_execute(선형)와 구조가 달라(classify→분기) 인터페이스가
  특정 형태에 안 새는 걸 노드열로 재확인. 한 실행에 한 분기만 타임라인에 뜸(질문→answer_a / 평서문→
  answer_b, 반대 분기 미발화)을 통합에서 실증.
- **3런 사다리 다 통과**: 단위 17단언(적합·정직 매니페스트·분기 결정성·드리프트0) + 통합(조건분기
  실 스트림) + codex 적대 no P0/P1(ctx 외 상태 없음·과대선언 없음·eval 경로 안 엶). 085/089 무회귀.
- **codex 스코프 격리 준수**: `.claude/skills/`는 codex가 읽으면 안 되므로 리뷰 번들에서 SKILL.md를
  빼고 **코드 파일만**(route.py·__init__·runtime 배선) 넘겼다. 문서는 보안 검토 대상이 아니라 손실 0.

## 아팠던 것 / 배운 것

- **분기 로직은 모듈 수준 순수함수로 빼야 모델 없이 검증된다**: `classify_route(text)->"a"/"b"`를
  노드 밖 순수함수로 두니 단위 테스트가 실 LLM 없이 분기 결정성을 단언. 노드 클로저 안에 묻었으면
  통합(모델 필요)에서만 확인 가능했다 — 스킬 규약에 "분기 로직 = 순수함수"로 명문화.
- **등록 반영엔 재기동 필요**: register는 import 시점 1회라 새 flow는 API 재기동 후 반영. 저작
  워크플로우로 허용하되 스킬·스펙에 명시(런타임 동적 로딩은 비목표 — 신뢰경계 우선).
