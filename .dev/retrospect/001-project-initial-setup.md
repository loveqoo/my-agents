# 001 — 프로젝트 초기 셋팅 루프 회고

날짜: 2026-06-19

## 루프 개요
- **목표:** `my-agents` 저장소의 프로젝트 초기 셋팅.
- **단계 흐름:**
  - 1 Scaffolding — 역할별 폴더 구조 + 운영 매뉴얼(CLAUDE.md) 수립
  - 2 Context — User/Project/Folder 스코프 기록 방식 합의, 단계 표기 규칙 추가
  - 3·4·5 Planning/Execution/Verification — **합의 스킵**
    - 사유: 설계·구현·검증할 신규 산출물이 없고, 클로드코드 설정은 이미 완료됨.
  - 6 Compounding — 본 회고 및 learning 자산화

## 무엇이 잘못됐나 (실수)
1. **User 스코프 과잉 해석 — 위치 선택을 상시 쓰기 허가로 착각** ([[002]])
   - Context 단계에서 사용자는 User 스코프 *기록 위치*로 "경로 기반 메모리"를 골랐을 뿐인데,
     나는 이를 "User 스코프에 언제든 써도 됨"으로 확대 해석했다.
   - 게다가 6단계 루프·주력 저장소 같은 **프로젝트 한정 내용을 cross-project여야 할 User 스코프에 섞어** 넣었다.
   - 사용자 지적: "User 스코프는 허락할 때만 접근해. 이 프로젝트 내용으로 User 스코프를 더럽히지 말라."
2. **회고 산출물 위치 혼동 — AI 회고를 인간 영역(docs/)에 쓰려 함**
   - 6단계에서 회고를 `docs/retrospect/`에 쓰려 했다.
   - `docs/`는 인간 영역(참고 전용). **AI 자신의 회고는 `.dev/`에 남겨야** 한다.
   - 사용자 지적: "너의 회고는 .dev에 기록하도록 해."
3. **영속성 모델을 늦게 인지** ([[001]])
   - 메모리가 프로젝트 절대경로를 키로 저장된다는 점을 폴더 리네임 맥락에서야 파악했다.
   - 경로가 바뀌면 기존 메모리가 고아가 되므로, 살아남아야 할 맥락은 저장소 안에 둬야 했다.

## 무엇을 배웠나 / 자산화
- **스코프 경계를 지킨다:** User 스코프 = cross-project(사용자 정체성·선호)만. 프로젝트 한정 맥락은
  저장소(`CLAUDE.md`, `.dev/`, `docs/`)에 둔다. 의심되면 저장소가 기본값. → [[002]]
- **영역 구분:** `docs/`는 인간 영역(참고 전용). **AI의 흔적·회고·학습은 `.dev/`.**
- **이식 가능한 맥락은 저장소에:** 경로 기반 메모리가 아니라 git으로 보존되는 저장소에 남긴다. → [[001]]
- **단계 표기 규칙 도입:** 매 응답 머리에 `[단계 N — 이름]`을 붙여 사람과 AI가 루프 위치를 공유한다.
  (CLAUDE.md 빠른 합의 규칙에 반영)

## 다음에 다르게 할 것 / 장치
- User 스코프에 쓸 일이 생기면 **먼저 "여기에 적어도 될까?"를 묻고**, 내용이 정말 cross-project인지 검증한다.
- 산출물을 쓰기 전에 **영역을 먼저 판단한다:** 인간 참고용 → `docs/`(명령 필요), AI 흔적 → `.dev/`.
- 같은 실수가 반복되면 Scaffolding 단계에서 훅/장치로 막는 것을 검토한다.
