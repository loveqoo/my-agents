# 004 — Web UI 프롬프트 확인 (read-only)

상태: **승인됨 (실행)**
날짜: 2026-06-19
연동 대상: [003-admin-web-ui-chat.md](./003-admin-web-ui-chat.md)

> web UI에서 **(a) 선택 에이전트의 페르소나**와 **(b) 모델로 실제 전송되는 메시지**를 읽기 전용으로 확인.

---

## 1. 목표 / 비범위

### 목표
- 선택한 에이전트의 **페르소나(시스템 프롬프트)** 를 화면에서 본다.
- 채팅 시 **모델로 실제 전송되는 메시지**(system + 대화)를 본다.

### 비범위
- 편집(읽기 전용 — 수정은 API로). 토큰 수/비용 표시. 백엔드 변경.

---

## 2. 핵심 판단 (왜 백엔드 변경 없이 되나)
- 서버는 `create_react_agent(prompt=persona)`로 **persona를 system 메시지로 prepend**한 뒤 대화 messages를 붙인다(툴 없음 = 추가 스키마 없음).
- 따라서 **실제 전송 메시지 = `[{role:"system", content: 페르소나}] + 대화 messages`**.
- 프론트는 persona(에이전트 객체에 포함)와 messages(브라우저 히스토리)를 모두 갖고 있어 **정확히 재구성** 가능 → 백엔드 무변경.

---

## 3. 결정 요약 (초안)
| 항목 | 결정 |
|---|---|
| 확인 대상 | 페르소나 + 실제 전송 메시지(system + 대화) |
| 편집 | 없음 (읽기 전용) |
| 구현 위치 | `admin/` 프론트만 (백엔드 무변경) |
| 표시 | 접이식 패널(Antd `Collapse`), 대화 진행 시 갱신 |

---

## 4. UI
- **2단 레이아웃: `|채팅영역|디버깅영역|`** (세로로 길어지는 단일 컬럼의 사용성 문제 해소).
- **좌(채팅):** 메시지 목록(고정 높이 스크롤, 자동 하단 스크롤) + 입력창.
- **우(디버깅, 읽기 전용):**
  - **페르소나(system):** 선택 에이전트의 `persona` 그대로.
  - **전송 메시지:** `[system: 페르소나] + 현재 대화 messages`를 role/content로 나열. messages 변경 시 갱신.

---

## 5. 검증 (완료 기준)
- [ ] 에이전트 선택 시 패널에 **페르소나**가 보인다.
- [ ] 대화하면 **전송 메시지**에 `system(페르소나) + 누적 대화`가 반영된다.
- [ ] 읽기 전용(수정 불가).
- [ ] 타자 비판 검증(codex) 통과.

---

## 6. 미해결 / 추후
- 서버 구성이 복잡해지면(툴 스키마/템플릿 주입 등) **백엔드 `POST /agents/{id}/preview`** 로
  실제 전송 페이로드를 서버 진실로 노출 — 현재는 불필요.
- 토큰 수/컨텍스트 길이 표시.
