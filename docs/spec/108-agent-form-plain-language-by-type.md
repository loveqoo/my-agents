# 108 — 에이전트 폼 재구성: 유저 언어 + 종류가 설정을 가른다

## 배경 / 왜

스펙 106·107로 능력 UI를 냈지만 사용자가 세 번째로 지적:
- **"impl" 같은 내부 단어를 쓰지 마라.** ("브로커·위임·능력·오케스트레이터"도 마찬가지.)
- **MCP를 두 곳에서 설정할 수 있어 유저가 이해 못 한다.** (직접 도구 칸 + 능력 위임 칸)
- **"첫 매치 위임" 같은 표현이 어렵다.** 단어·UX가 직관적이지 못하다.

**뿌리 원인**: 내부 아키텍처(impl 레지스트리·브로커·능력 kind)를 그대로 UI 단어로 노출했다. 게다가
런타임은 종류에 따라 **다른 것만** 쓰는데(아래) 폼은 둘 다 보여줘 중복이 생겼다.

## 아키텍처 근거 (코드 확인)

- **직접 응답 에이전트**(`DefaultUiAgent`, impl 미선언): `ctx.tools`(mcps+RAG)+persona+기억을 **직접**
  써서 답한다. `ctx.broker`를 **안 쓴다**.
- **조율형 에이전트**(impl=`orchestrate`): `ctx.broker`(=capabilities로 스코프)**만** 써서 다른 곳에
  위임한다. `ctx.tools`(직접 mcps/RAG)를 **안 쓴다**(orchestrate.py는 broker.discover/invoke만 호출).

→ 두 설정면은 **종류에 따라 상호 배타**다. 같은 화면에 둘 다 두면 반드시 중복(MCP가 두 번)이 생긴다.

## 설계

### 1. 유저 언어 (내부어 제거)
- "실행 방식 (impl)" → **"에이전트 종류"**. 선택지 2개(사용자 확정: AskUserQuestion):
  - **직접 응답**(내부 `impl=""`) — "스스로 도구·문서·기억을 써서 답합니다."
  - **조율형**(내부 `impl="orchestrate"`) — "일을 다른 에이전트·도구에 넘겨 처리합니다."
- "능력 (브로커 위임)" → **"무엇에 맡길까요?"**. kind 라벨도 유저어:
  `내 기억`→**사용자 기억**, `문서 컬렉션 (RAG)`→**문서**, `MCP 서버`→**도구**, `다른 에이전트 (A2A)`→**다른 에이전트**.
- cap 항목 라벨에서 **기술 접미사 제거**(`(memory:user)`·`(rag:docs)`·`(mcp:...)`·`(agt_...)`) — 표시엔
  사람이 읽는 이름만, 내부 id는 값으로만 유지.
- "첫 매치 위임"·"랭킹 조합" 등 **전략 변형은 UI에서 숨김** — 조율형은 기본 전략(`orchestrate`) 사용.
  (구 저장분이 `orchestrate_ranked` 등이면 값 보존해 "조율형"으로 표시, 편집 시 안 사라지게.)

### 2. 종류가 설정면을 가른다 (중복 제거)
- **공통**(둘 다): 이름·모델·페르소나·Temperature·채팅 히스토리·대화 저장.
- **직접 응답만**: 메모리 타입·지식 소스(RAG)·권한·도구(MCP) — 직접 쓰는 자원.
- **조율형만**: "무엇에 맡길까요?" 한 칸(다른 에이전트·도구·문서·사용자 기억). 직접 자원 칸은 **숨김**
  → MCP·문서가 **정확히 한 곳**에만 나온다(중복 소멸).

### 3. 배선 (백엔드 무변경)
- 저장 모델(`impl`·`capabilities`)·왕복은 106 그대로. 이번은 **표현 계층만**(라벨·조건부·순서).
- 프론트 `/agent-impls` fetch/impls state/impls prop 제거(종류 선택지는 2개 고정) — 백엔드 엔드포인트는
  유지(무해, 후속 활용 여지). 종류 selector는 레지스트리 대신 사용자어 2택으로 단순화.

## 검증

- **브라우저(직접)**: (a) 종류=직접 응답 → 직접 자원 칸(메모리·문서·권한·도구) 보이고 "맡길" 칸 없음.
  (b) 종류=조율형 → "무엇에 맡길까요?"만 보이고 직접 자원 칸 사라짐(MCP 한 곳). (c) 화면 어디에도
  "impl·브로커·위임·오케스트레이터" 단어 없음. (d) cap 라벨에 `memory:user` 등 기술 id 안 보임.
  (e) 저장 왕복 무회귀(조율형+맡길 대상 저장→재열기 반영). 스크린샷.
- 무회귀: 백엔드 106 왕복(impl/caps 저장·직렬화) 무변경.

## 비목표 (OUT)

- 전략 변형(랭킹/조건 분기/계획-실행) UI 노출 — 필요 시 후속 "고급"에서 유저어로.
- MCP 툴 단위 granularity(106 유지).
- 직접 자원 칸(권한·도구) 자체의 접이식화 — 이번은 종류 분기·용어까지. 항목이 많아지면 후속.
