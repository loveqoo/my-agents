# 107 — 능력 피커 UX: 오케스트레이터 조건부 + 종류별 접이식

## 배경 / 왜

스펙 106이 능력 피커를 **종류별 체크박스 벌판**으로 냈다. 사용자가 화면을 보고 지적:
"설정화면이 사용자 편의성을 많이 해친다. RAG가 늘면 폼이 계속 복잡해지고 어렵게 느껴진다."
정확한 지적 — 컬렉션/MCP/에이전트가 늘수록 폼이 세로로 무한정 길어지고, 처음 보는 사람에겐
체크할 것이 압도적이다.

## 목표 (사용자 확정 방향)

두 축을 함께 고쳐 폼을 단순하게:
1. **능력 칸은 오케스트레이터 impl(`orchestrate`·`orchestrate_ranked`)일 때만 표시.** 보통
   에이전트 생성 시엔 능력 칸이 **아예 안 보임**(대다수 경우 화면 단순화). 106에서 OUT으로 미뤘던
   조건부 표시를 이 지적으로 당김.
2. **종류별 접이식(accordion).** 종류(내 기억/문서 컬렉션/MCP/다른 에이전트)를 **기본 접힘**
   패널로. 헤더에 `선택수/전체수` 배지. 펼친 종류만 목록 노출 → 늘어도 접혀 있어 짧다.
   (사용자 선택: AskUserQuestion "종류별 접이식".)

## 설계

- `Collapse`(antd) 사용. 각 종류 = 한 패널.
  - 헤더: `<종류 라벨> · <선택수>/<전체수>` (예: `문서 컬렉션 (RAG) · 1/12`).
  - 본문: 항목이 많으면(> 6) **패널 내 검색 입력**으로 필터(RAG/MCP/agent처럼 늘 수 있는 종류),
    그 아래 체크박스 목록. `내 기억`(고정 2개)은 검색 없음.
- **기본 펼침 규칙**: 이미 선택이 있는 패널만 펼쳐 연다(편집 재열기 시 무엇이 켜졌는지 바로 보임).
  선택 0인 패널은 접힘.
- **조건부 표시**: `isOrchestratorImpl(form.impl)`가 참일 때만 능력 Field 렌더. 아니면 Field 자체 없음
  → 106의 "오케스트레이터에서 사용됨" 힌트 Tag 제거(칸이 아예 없으니 불필요).
- 상태: 패널별 검색어 `capSearch: Record<string,string>`(폼 컴포넌트 로컬). 선택 자체는 기존
  `form.capabilities`·`toggleCap` 그대로(데이터 모델·저장 배선 무변경).

## 검증

- **브라우저(직접)**: (a) impl=기본일 때 능력 칸 없음, impl=orchestrate로 바꾸면 나타남. (b) 종류
  패널 기본 접힘, 헤더 카운트 정확, 펼치면 목록·검색. (c) RAG를 여러 개로 늘려도(테스트 컬렉션 시드)
  폼 높이 안정 — 접힌 상태 유지. (d) 선택 있는 패널 기본 펼침(편집 모드). (e) 토글·저장 왕복 무회귀.
  스크린샷 증거.
- 무회귀: 106 백엔드/왕복은 무변경(UI만) — 기존 verify 유지.

## 비목표 (OUT)

- MCP 툴 단위 granularity(106 OUT 유지).
- 능력 카탈로그 서버 검색(클라이언트 필터로 충분 — 목록은 이미 로드됨).
- 다른 폼 섹션(권한·MCP·메모리)의 접이식화 — 이번은 능력 칸만(지적 범위). 필요하면 후속.
