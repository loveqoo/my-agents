# 에이전트 만들기 — 페르소나·모델·도구·메모리

에이전트는 페르소나, 파운데이션 모델, 도구(MCP), 메모리 네 가지 구성으로 정의됩니다.
이 문서는 새 에이전트를 만들 때 각 구성을 어떻게 채우는지 설명합니다.

## 페르소나
페르소나는 에이전트의 역할과 해야 할 업무를 정의합니다. "고객지원 상담원", "리서치
어시스턴트"처럼 구체적으로 적을수록 응답 품질이 좋아집니다.

## 파운데이션 모델
에이전트가 동작할 모델을 프로바이더·모델 메뉴에서 등록한 것 중에서 고릅니다. 채팅용
모델(kind=chat)을 선택해야 하며, 임베딩 모델은 RAG 컬렉션에만 쓰입니다.

## 도구와 메모리
필요한 MCP 도구를 연결하면 에이전트가 외부 기능을 호출할 수 있습니다. 메모리는 작업
기억과 사용자에 대한 장기 기억으로 나뉘며, Mem0가 이를 관리합니다.

## 버전과 활성화
에이전트는 버전으로 관리됩니다. 초안(draft)을 만들고 검토한 뒤 활성(active)으로 올리며,
지난 버전은 보관(archived)됩니다. 활성 버전이 실제 대화에 사용됩니다.
