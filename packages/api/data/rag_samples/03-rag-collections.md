# RAG 컬렉션 — 문서 적재와 의미 검색

RAG 컬렉션은 에이전트가 답변할 때 참고하는 지식베이스입니다. 문서를 업로드하면 텍스트가
청크로 분할되고 임베딩되어 pgvector에 저장됩니다. 질의가 들어오면 의미적으로 가까운 청크를
찾아 근거로 사용합니다.

## 임베딩 모델이 먼저 필요합니다
컬렉션을 만들려면 임베딩 모델이 최소 하나 등록되어 있어야 합니다. 임베딩 모델이 없으면 RAG
메뉴의 동작이 비활성화되며, 먼저 프로바이더·모델에서 임베딩 모델을 등록해야 합니다. 컬렉션의
임베딩 모델과 차원은 생성 후 변경할 수 없습니다.

## 빈 컬렉션의 의미
임베딩 모델이 있는 빈 컬렉션은 고장이 아니라 문서 업로드를 기다리는 정상 상태입니다. 문서를
올리면 상태가 ingesting을 거쳐 ready로 바뀌고 문서 수와 청크 수가 채워집니다.

## 청크 크기와 겹침
청크 크기는 한 조각의 최대 길이를, 겹침은 인접 조각이 공유하는 길이를 정합니다. 너무 크면
검색이 뭉툭해지고, 너무 작으면 문맥이 끊깁니다. 기본값은 크기 1000, 겹침 200입니다.
